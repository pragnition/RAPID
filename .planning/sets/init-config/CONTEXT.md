<domain>
## Set Boundary
This set addresses four related gaps in RAPID's initialization and configuration layer:

1. **DEFINITION.md generation** -- Init skill generates CONTRACT.json for each set but never creates DEFINITION.md, causing loadSet() to crash with ENOENT
2. **Graceful loadSet() fallback** -- loadSet() in plan.cjs should handle missing DEFINITION.md without crashing
3. **Project-wide solo mode config** -- A `solo` field in config.json so single-developer projects default to solo mode
4. **Worktree package management** -- createWorktree() should detect package manager and install dependencies
5. **Auto-commit after init** -- Init skill should auto-commit all generated planning artifacts

All changes are defensive middleware -- additive guards at failure boundaries without restructuring the 5-layer architecture (Skill -> Agent -> CLI -> Library -> Storage).
</domain>

<decisions>
## Implementation Decisions

### loadSet() Fallback Strategy

- Return `{ definition: null, contract }` when DEFINITION.md is missing (not empty string or stub)
- All 6 callsites (worktree.cjs, plan.cjs, merge.cjs x2, stub.cjs x2, execute.cjs) must be audited for null-safe access
- Emit a `console.error` warning with set name and missing file path when DEFINITION.md is absent
- loadSet() must never throw ENOENT for DEFINITION.md; only throw for missing set directory or CONTRACT.json

### Solo Mode Config Behavior

- Read solo config from `.planning/config.json` at the top level: `{ solo: true }`
- Config provides the default; `--solo` CLI flag still works as an explicit override
- No `--no-solo` flag needed -- if config says solo:true but user omits --solo, normal (worktree) mode is used
- Solo config must be checked in start-set skill and set-init CLI command

### Worktree Dependency Install

- createWorktree() should auto-install dependencies after worktree creation
- Detect package manager by checking for lockfiles (package-lock.json -> npm, yarn.lock -> yarn, pnpm-lock.yaml -> pnpm)
- Use standard full install (no --prefer-offline or symlink) for maximum reliability
- Install runs synchronously after worktree creation so node_modules is immediately available

### Auto-Commit Scope

- Only commit files under `.planning/` (ROADMAP.md, STATE.json, config.json, contracts, definitions)
- Never commit user source files or files outside .planning/
- Commit message format: `rapid:init({project-name}): initialize project planning artifacts`
- Check git status before committing; if working tree has uncommitted changes, warn the user but still commit only .planning/ files (user's changes remain unstaged)

### Claude's Discretion

- DEFINITION.md generation approach in the init skill (CLI layer vs skill layer)
- Package manager detection implementation details
- Error handling for failed dependency installs (warn vs fail)
</decisions>
