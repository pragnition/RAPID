v3.0 Command Structure (7 core + 4 auxiliary):

CORE LIFECYCLE:
1. /rapid:init -- Initialize project (research, roadmap, set decomposition)
2. /rapid:start-set -- Create isolated worktree, generate scoped CLAUDE.md
3. /rapid:discuss-set -- Capture developer implementation vision
4. /rapid:plan-set -- Plan all waves in a set (research, wave plans, validation)
5. /rapid:execute-set -- Execute waves (one executor agent per wave, sequential)
6. /rapid:review -- Unit test + bug hunt + UAT pipeline
7. /rapid:merge -- Merge completed sets to main with conflict detection

AUXILIARY:
1. /rapid:status -- Project dashboard with set statuses
2. /rapid:install -- Plugin installation, shell configuration
3. /rapid:new-version -- Start new milestone/version cycle
4. /rapid:add-set -- Add sets mid-milestone

KEPT UTILITIES (not part of 7+4):
- /rapid:assumptions, /rapid:pause, /rapid:resume, /rapid:context, /rapid:cleanup, /rapid:quick, /rapid:help

DEPRECATED (show migration messages):
- /rapid:set-init -> /rapid:start-set
- /rapid:discuss -> /rapid:discuss-set
- /rapid:execute -> /rapid:execute-set
- /rapid:plan -> /rapid:plan-set or /rapid:init
- /rapid:wave-plan -> /rapid:plan-set
- /rapid:new-milestone -> /rapid:new-version

v3.0 Architecture:
- Sets are the unit of parallel work (one per developer)
- No user-visible wave/job hierarchy -- execution is per-wave internally but the user just runs execute-set
- Interface contracts (CONTRACT.json) replace GATES.json
- Skills are their own orchestrators (no central orchestrator agent)
- 26 agents total (5 core hand-written + 21 generated)

v3.0 Agent Dispatch:
- init: 6 researchers + synthesizer + roadmapper + codebase-synthesizer
- start-set: set-planner
- plan-set: research-stack + planner + plan-verifier
- execute-set: executor (one per wave) + verifier
- review: scoper + unit-tester + bug-hunter + devils-advocate + judge + bugfix + uat
- merge: set-merger (per set) + conflict-resolver (per conflict)
- quick: planner + plan-verifier + executor
