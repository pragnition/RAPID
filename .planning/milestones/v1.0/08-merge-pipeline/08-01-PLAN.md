Create `rapid/src/lib/merge.cjs` following the established library module pattern (see execute.cjs for reference). The module depends on contract.cjs, dag.cjs, worktree.cjs, execute.cjs, and plan.cjs.

**Implementation details:**

1. **runProgrammaticGate(cwd, setName)**
   - Load set data via `plan.loadSet(cwd, setName)`
   - Validate contract schema: `contract.compileContract(setData.contract)` -- returns { valid, errors }
   - Generate and run contract tests: write temp test file via `contract.generateContractTest(setData.contract, setName)`, run with `execSync('node "' + testFile + '"', { cwd, stdio: 'pipe', timeout: 30000 })` (use plain `node` per 07-02 decision -- avoids nested TAP conflicts)
   - Load registry to get worktree path: `worktree.loadRegistry(cwd)`
   - Get changed files: `execute.getChangedFiles(worktreePath, baseBranch)` -- use `worktree.detectMainBranch(cwd)` for baseBranch
   - Load OWNERSHIP.json from `.planning/sets/OWNERSHIP.json`
   - For each changed file, call `contract.checkOwnership(ownershipData.ownership, file)`. If owner !== null and owner !== setName, check CONTRIBUTIONS.json at `.planning/sets/{setName}/CONTRIBUTIONS.json` for exception. If no exception, add to ownershipViolations
   - Return { passed: (contractValid && testsPass && ownershipViolations.length === 0), contractValid, testsPass, ownershipViolations, testOutput }

2. **prepareReviewContext(cwd, setName)**
   - Load set via plan.loadSet, registry via worktree.loadRegistry
   - Get changed files, contract string, ownership data, definition
   - Return lean context object

3. **assembleReviewerPrompt(cwd, setName, programmaticResults)**
   - Follow the assembleExecutorPrompt pattern from execute.cjs
   - Build prompt string with sections: Merge Review heading, Changed Files list, Contract JSON, Programmatic Validation Results (pass/fail for each gate), Review Instructions
   - Review instructions tell the agent to produce a REVIEW.md with sections and a <!-- VERDICT:X --> marker

4. **writeReviewMd(setDir, reviewData)**
   - Write Markdown file at `path.join(setDir, 'REVIEW.md')` with:
     - `# Review: {setName}` heading
     - `**Reviewed:** {ISO timestamp}`, `**Verdict:** {verdict}`
     - `<!-- VERDICT:{verdict} -->` HTML comment on its own line
     - Sections: Contract Validation, Ownership Check, Test Results, Findings (Blocking, Fixable, Suggestions)
   - Each findings section is a bulleted list; empty sections show "None"

5. **parseReviewVerdict(setDir)**
   - Read REVIEW.md from setDir
   - Find `<!-- VERDICT:` pattern via regex `/<!-- VERDICT:(APPROVE|CHANGES|BLOCK) -->/`
   - Return { verdict, found: true } or { found: false }

6. **getMergeOrder(cwd)**
   - Read `.planning/sets/DAG.json`, parse JSON
   - Call `dag.getExecutionOrder(dagJson)` -- returns [['set-a','set-b'], ['set-c']]
   - Return the wave-grouped arrays directly

7. **mergeSet(projectRoot, setName, baseBranch)**
   - `worktree.gitExec(['checkout', baseBranch], projectRoot)` -- if not ok, return failure
   - `worktree.gitExec(['merge', '--no-ff', 'rapid/' + setName, '-m', 'merge(' + setName + '): merge set into ' + baseBranch], projectRoot)` -- merge commit per user decision
   - On failure: check stderr for 'CONFLICT' or 'Automatic merge failed', run `gitExec(['merge', '--abort'], projectRoot)` if conflict, return { merged: false, reason, detail }
   - On success: get HEAD hash via `gitExec(['rev-parse', 'HEAD'], projectRoot)`, return { merged: true, branch: 'rapid/' + setName, commitHash }

8. **runIntegrationTests(projectRoot)**
   - `execSync('node --test rapid/src/lib/*.test.cjs', { cwd: projectRoot, stdio: 'pipe', timeout: 30000 })`
   - Return { passed: true, output } on success, { passed: false, output: stderr } on error
