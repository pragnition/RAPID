# Phase 09.2: Create Setup Script and Fix RAPID_TOOLS Paths - Context

**Gathered:** 2026-03-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Create a setup/install system that ensures RAPID_TOOLS env var is correctly set regardless of installation method (marketplace or git clone). Remove hardcoded fallback paths from all SKILL.md and core module files. Provide both a CLI setup.sh script and a /rapid:install skill.

</domain>

<decisions>
## Implementation Decisions

### Setup script scope
- Full bootstrap: set RAPID_TOOLS, validate Node.js/git prereqs, run npm install, verify plugin is functional
- Auto-register as Claude Code plugin (runs `claude plugin add .` if not already registered)
- Lives at repo root: `setup.sh`
- Idempotent: re-running skips already-completed steps, safe for updates and troubleshooting

### Path fallback strategy
- Remove all fallbacks: change `${RAPID_TOOLS:-$HOME/RAPID/src/bin/rapid-tools.cjs}` to just `${RAPID_TOOLS}`
- When RAPID_TOOLS is not set: print clear error message with link to installation instructions and exit 1
- All references consistent: SKILL.md files AND core module files (core-state-access.md, core-context-loading.md) all use bare `$RAPID_TOOLS`

### Installation flow
- Primary method: Plugin marketplace (`claude plugin add fishjojo1/RAPID`)
- Secondary method: Git clone + `./setup.sh`
- New `/rapid:install` skill that runs setup steps internally from within Claude Code
- Install skill auto-detects whether running from clone or marketplace install
- Both installation methods supported equally by the install skill
- Marketplace install path: `~/.claude/plugins/rapid/`

### Shell compatibility
- Support bash, zsh, and fish
- Auto-detect user's shell via $SHELL env var
- Persist RAPID_TOOLS: append to shell config file (.bashrc, .zshrc, config.fish)
- At runtime, ASK user which persistence method they prefer (shell config append recommended, or dedicated env file)
- Set RAPID_TOOLS in current session AND persist to config (no terminal restart needed)

### Claude's Discretion
- Exact setup.sh script structure and step ordering
- Error message wording and formatting
- How to detect marketplace vs clone installation
- npm install strategy (if package.json has deps)
- How to handle edge cases (missing shell config files, read-only home dirs)

</decisions>
