    6. Create `rapid/src/lib/prereqs.cjs`:
       - `checkTool({ name, command, parseVersion, minVersion, required, reason })`: Execute command via `child_process.execSync` with stdio: 'pipe', timeout: 5000, encoding: 'utf-8'. Parse version from output. Compare with minVersion. Return `{ name, status, version, minVersion, required, reason, message }`. Return ALL results regardless of individual failures.
       - `checkGitRepo(cwd)`: Run `git rev-parse --show-toplevel` with cwd. Return `{ isRepo, toplevel }`.
       - `formatPrereqSummary(results)`: Build markdown table from results array. Return `{ table, hasBlockers, hasWarnings }`.
       - Export all: `{ validatePrereqs, checkGitRepo, formatPrereqSummary, checkTool, compareVersions }`.

       - If `args` includes `--git-check`: Run `checkGitRepo(process.cwd())`, output JSON `{ isRepo, toplevel }`, return.
       - If `args` includes `--json`: Run `validatePrereqs()`, output raw JSON array.

    3. Update the USAGE string at the top of rapid-tools.cjs to include:
       ```

       prereqs --git-check    Check if current directory is a git repository
       prereqs --json         Output raw prerequisite results as JSON
       ```

    4. Note: The `prereqs` subcommand does NOT need `findProjectRoot()` -- it operates on cwd directly since it runs before any project exists. Add a special case in main() so the prereqs command skips the `findProjectRoot()` call. Pattern:

       ```javascript
         handlePrereqs(args.slice(1));
         return;
       }
       // All other commands need project root
       let cwd;
       try { cwd = findProjectRoot(); } catch ...
       ```

    IMPORTANT: The prereqs command must work BEFORE `.planning/` exists (it runs during init, which creates `.planning/`). That's why it must bypass `findProjectRoot()`. Use `~` for home directory paths in shell commands.
