Rewrite skills/install/SKILL.md to own all user-facing interaction. Add AskUserQuestion to the allowed-tools frontmatter line (alongside Read and Bash).

**Step 0: Detect Installation Location** (keep existing logic unchanged)
- Check CLAUDE_SKILL_DIR for marketplace vs clone
- Set RAPID_ROOT

**Step 1: Run Non-Interactive Bootstrap**
- Run `bash "$RAPID_ROOT/setup.sh"` using the Bash tool
- This handles prereqs, npm install, validate, .env write, and plugin register
- If setup.sh fails: use AskUserQuestion with header "Setup failed", options: "Retry setup" (re-run setup.sh) / "Show manual steps" (display manual install commands) / "Cancel installation"

**Step 2: Detect Shell and Select Config File (INST-01)**
- Read $SHELL env var to detect shell: `SHELL_NAME=$(basename "$SHELL")`
- Display: "Detected shell: {SHELL_NAME} ({$SHELL})"
- Map shell to config files:
  - bash -> ~/.bashrc (prefer) or ~/.bash_profile
  - zsh -> ~/.zshrc
  - fish -> ~/.config/fish/config.fish
  - unknown -> ~/.profile
- Check if RAPID_TOOLS already exists in any config file (grep for RAPID_TOOLS in detected config). If found, display "RAPID_TOOLS already configured in {file}, skipping shell config" and jump to Step 4 (Verify).
- If not already configured: use AskUserQuestion:
  - Header: "Shell configuration"
  - Text: "Detected shell: {SHELL_NAME} ({$SHELL}). Choose where to persist the RAPID_TOOLS environment variable:"
  - Options (show only existing files, mark detected shell's config as recommended):
    - "{config_file} (recommended)" -- for the detected shell's config file
    - Other existing config files as additional options
    - "Skip -- use .env only" with description: "RAPID will work in Claude Code via .env fallback. Shell config is for terminal usage."

**Step 3: Write Shell Config and Auto-Source (INST-02)**
- If user chose a config file:
  - Determine export line: fish uses `set -gx RAPID_TOOLS "{path}"`, others use `export RAPID_TOOLS="{path}"`
  - Append export line to chosen config file (with `# RAPID plugin for Claude Code` comment)
  - Auto-source and verify in ONE Bash tool call:
    - For fish: `fish -c 'source ~/.config/fish/config.fish; echo $RAPID_TOOLS'` then `node` with that path + `prereqs`
    - For bash: `bash -c 'source ~/.bashrc && echo $RAPID_TOOLS && node "$RAPID_TOOLS" prereqs'`
    - For zsh: `zsh -c 'source ~/.zshrc && echo $RAPID_TOOLS && node "$RAPID_TOOLS" prereqs'`
  - The RAPID_TOOLS path is: `$RAPID_ROOT/src/bin/rapid-tools.cjs` (derived from RAPID_ROOT set in Step 0)
  - If source + verify succeeds: display "Shell config updated and verified. RAPID_TOOLS is active."
  - If source + verify FAILS (INST-03): display fallback guidance:
    - Show exact source command in a code block: `` `source ~/.zshrc` `` (or shell equivalent)
    - Note: "Note: .env fallback is already configured -- RAPID will work in Claude Code regardless. The shell config is for terminal usage."
    - Use AskUserQuestion with header "Auto-source failed", options: "Retry" (re-run source+verify) / "Continue anyway" (proceed to Step 4 since .env works) / "Cancel installation"
- If user chose "Skip": display "Skipped shell config. RAPID_TOOLS loaded from .env in Claude Code sessions." and proceed to Step 4.

**Step 4: Verify Installation**
- Load RAPID_TOOLS from .env if not set: `export $(grep -v '^#' "$RAPID_ROOT/.env" | xargs)`
- Run `node "${RAPID_TOOLS}" prereqs` to verify full tool chain
- If prereqs fails: AskUserQuestion with header "Prereqs verification failed", options: "Retry prereqs" / "Show manual fix steps" / "Cancel"

**Step 5: Post-Install Next Actions**
- AskUserQuestion with header "Installation complete":
  - "Run /rapid:help" with description "See all available RAPID commands and workflow guidance"
  - "Run /rapid:init" with description "Initialize planning infrastructure for a new project"
  - "Done" with description "Exit installer"

**Global rules:**
- No STOP or halt keywords anywhere
- Use tilde ~ in display text for paths (per CLAUDE.md: use ~ instead of $HOME for shell commands)
- All error paths use AskUserQuestion with retry/skip/cancel pattern
