Read the current skills/plan/SKILL.md and skills/init/SKILL.md (for AskUserQuestion pattern reference).

Modify plan SKILL.md with these specific changes:

1. **Frontmatter**: Add AskUserQuestion to the allowed-tools list.

2. **Step 1 (existing sets detected)**: Replace the numbered text options with AskUserQuestion:
   - Header: "Existing sets"
   - Option 1: "Re-plan" with description explaining it deletes .planning/sets/ and .planning/contracts/ and starts fresh decomposition
   - Option 2: "View current" with description explaining it shows current set details for review
   - Option 3: "Cancel" with description explaining it exits without changes
   - After "View current" is selected: display each set's details using `node "${RAPID_TOOLS}" plan load-set <name>`, then show a SECOND AskUserQuestion with header "Continue" and options:
     - "Re-plan" (same description as above)
     - "Cancel" (exits without changes)
   - This ensures View is a preview step, not a dead end.

3. **Step 4 (proposal review)**: Replace the numbered text options with AskUserQuestion:
   - Header: "Plan proposal"
   - Option 1: "Approve" with description that includes an inline summary of the proposed sets (e.g., "Create {N} sets across {M} waves: {set1} || {set2} > {set3}"). The executor should build this summary string from the proposal data before calling AskUserQuestion.
   - Option 2: "Modify" with description explaining the developer can describe requested changes
   - Option 3: "Cancel" with description explaining it aborts planning
   - When "Modify" is selected: use a plain text prompt (not AskUserQuestion) to ask what changes are needed, then pass feedback to subagent and loop back to Step 4. This matches Phase 10's "structured gate then freeform" pattern.

4. **STOP removal**: Replace all instances of `**STOP**` with appropriate exit behavior:
   - "View current" after second gate Cancel: display "Planning cancelled. Run `/rapid:plan` to resume." and end the skill.
   - "Cancel" at Step 1: display "Planning cancelled." and end the skill.
   - "Cancel" at Step 4: display "Planning cancelled. Your existing sets are unchanged." and end the skill.
   - Use clear exit text, not STOP keyword.

5. Preserve all other steps (2, 3, 5, 6) and Important Notes unchanged.

  <name>Task 2: Rewrite assumptions SKILL.md with AskUserQuestion at Steps 1 and 4</name>
  <files>skills/assumptions/SKILL.md</files>
  <action>
Read the current skills/assumptions/SKILL.md.

Modify assumptions SKILL.md with these specific changes:

1. **Frontmatter**: Add AskUserQuestion to the allowed-tools list (currently only has Read, Bash).

2. **Step 1 (set selection when no set name provided)**:
   - After parsing `availableSets` from JSON output:
   - **If <=4 sets**: Use AskUserQuestion with header "Select set" and each set name as an option label. Add an "Other" option for typing a set name manually. No consequence descriptions needed since options are just names.
   - **If >4 sets**: Fall back to a numbered text list with freeform input (current behavior). This avoids overwhelming AskUserQuestion with too many options.
   - **If no sets**: Display error message "No sets have been defined yet. Run `/rapid:plan` first to decompose the project into sets." and end the skill (remove STOP keyword).

3. **Step 4 (developer feedback)**: Replace the numbered text options with AskUserQuestion:
   - Header: "Assumptions"
   - Option 1: "Correct assumptions" with description "Describe what needs to change — you will need to re-run /rapid:plan to modify set definitions"
   - Option 2: "Note for execution" with description "Add notes to the set's DEFINITION.md for the executor to see during implementation"
   - Option 3: "Looks good" with description "Assumptions are correct, proceed with confidence"
   - When "Correct assumptions" is selected: use a plain text prompt (not AskUserQuestion) to ask what specifically is wrong. Then display the guidance about running /rapid:plan. Matches "structured gate then freeform" pattern.
   - When "Note for execution" is selected: suggest adding notes to `.planning/sets/{set-name}/DEFINITION.md` under `## Executor Notes` section.
   - When "Looks good" is selected: show a SECOND AskUserQuestion with header "Continue" and options:
     - "Review another set" with description "Go back to set selection"
     - "Done" with description "Finish reviewing assumptions"
   - If "Review another set": loop back to Step 1 set selection.
   - If "Done": display confirmation and end the skill.

4. **STOP removal**: Replace all `**STOP**` keywords:
   - No sets found: end with guidance text, no STOP.
   - Set not found error: end with available sets list, no STOP.
   - After corrections noted: end with /rapid:plan guidance, no STOP.
   - Use clear exit text throughout.

5. Preserve Steps 2 and 3, and Important Notes unchanged (except any STOP references in Important Notes).
