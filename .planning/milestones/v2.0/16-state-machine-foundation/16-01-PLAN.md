<task type="auto" tdd="true">
  <name>Task 1: Zod schemas for hierarchical state</name>
  <files>src/lib/state-schemas.cjs, src/lib/state-schemas.test.cjs</files>
  <behavior>
    - ProjectState.parse() accepts valid deeply nested hierarchy and returns defaults for missing optional fields
    - JobStatus enum only allows: pending, executing, complete, failed
    - WaveStatus enum only allows: pending, discussing, planning, executing, reconciling, complete
    - SetStatus enum only allows: pending, planning, executing, reviewing, merging, complete
    - JobState requires id and status, optional: startedAt, completedAt, commitSha, artifacts (defaults to [])
    - WaveState requires id and status, has jobs array (defaults to [])
    - SetState requires id and status, has waves array (defaults to [])
    - MilestoneState requires id and name, has sets array (defaults to [])
    - ProjectState requires version (literal 1), projectName, currentMilestone, lastUpdatedAt, createdAt; milestones defaults to []
    - Invalid data (wrong status values, missing required fields) produces ZodError with meaningful paths
    - safeParse returns success:false with error issues for invalid data
  </behavior>
  <action>
    RED: Write state-schemas.test.cjs first with tests covering all behaviors above. Tests import from state-schemas.cjs. Run tests -- they MUST fail (module doesn't exist yet).

    GREEN: Create state-schemas.cjs using Zod z.object, z.enum, z.array, z.literal, z.string, .default(), .optional(). Follow the schema structure from RESEARCH.md Pattern 1. Use 'use strict' and CommonJS module.exports. Export all schema objects AND status enums.

    Schema hierarchy:
    - JobStatus = z.enum(['pending', 'executing', 'complete', 'failed'])
    - JobState = z.object({ id: z.string(), status: JobStatus.default('pending'), startedAt: z.string().optional(), completedAt: z.string().optional(), commitSha: z.string().optional(), artifacts: z.array(z.string()).default([]) })
    - WaveStatus = z.enum(['pending', 'discussing', 'planning', 'executing', 'reconciling', 'complete'])
    - WaveState = z.object({ id: z.string(), status: WaveStatus.default('pending'), jobs: z.array(JobState).default([]) })
    - SetStatus = z.enum(['pending', 'planning', 'executing', 'reviewing', 'merging', 'complete'])
    - SetState = z.object({ id: z.string(), status: SetStatus.default('pending'), waves: z.array(WaveState).default([]) })
    - MilestoneState = z.object({ id: z.string(), name: z.string(), sets: z.array(SetState).default([]) })
    - ProjectState = z.object({ version: z.literal(1), projectName: z.string(), currentMilestone: z.string(), milestones: z.array(MilestoneState).default([]), lastUpdatedAt: z.string(), createdAt: z.string() })

    Run tests -- they MUST pass.
  </action>
  <verify>
    <automated>cd ~/Projects/RAPID && node --test src/lib/state-schemas.test.cjs</automated>
  </verify>
  <done>All schema tests pass. Schemas correctly validate the full hierarchy, reject invalid data with ZodError, and apply defaults for optional fields.</done>
