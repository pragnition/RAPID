  <name>Task 1: Rewrite init SKILL.md with full multi-agent pipeline</name>
  <files>skills/init/SKILL.md</files>
  <action>
    Complete rewrite of skills/init/SKILL.md implementing the full extended init flow per user decisions.

    **SKILL.md frontmatter:**
    - name: init
    - description: Initialize a new RAPID project with research and roadmap generation
    - allowed-tools: Bash(rapid-tools:*), Agent, AskUserQuestion, Read, Write, Glob, Grep

    **Load .env file** at the start of the skill per project convention (source .env if it exists for RAPID_TOOLS and other env vars).

    **Pipeline Steps (in order):**

    **Step 1: Prerequisites**
    Run `rapid-tools.cjs prereqs check` via Bash. If failures, show clear error with what to install. Stop if critical prereqs missing.

    **Step 2: Git Check**
    Verify current directory is a git repository. If not, offer to `git init`.

    **Step 3: Existing Detection**
    Run `rapid-tools.cjs init detect`. If .planning/ exists, use AskUserQuestion:
    - "Reinitialize" -- Back up and start fresh
    - "Upgrade" -- Add missing files, keep existing
    - "Cancel" -- Exit

    **Step 4: Setup Questions**
    Use AskUserQuestion for each (batch where possible):

    a) Project name (text input, default to directory name)
    b) Project description (text input)
    c) Team size: AskUserQuestion with options "1 developer", "2-3 developers", "4-5 developers", "6+ developers"
    d) Model selection: AskUserQuestion with options:
       - "Opus" -- "Higher quality, slower, more expensive. Best for complex projects."
       - "Sonnet" -- "Good balance of quality and speed. Recommended for most projects."

    **Step 5: Scaffold**
    Run `rapid-tools.cjs init scaffold --name {name} --description {desc} --team-size {N} --mode {mode}`.
    Run `rapid-tools.cjs init write-config --model {model} --team-size {N} --name {name}`.
    Run `rapid-tools.cjs init research-dir`.

    **Step 6: Brownfield Detection**
    Use Read/Grep to check if the project has existing source code (look for src/, lib/, app/, common language files).
    Alternatively run `rapid-tools.cjs context detect` if available.

    If brownfield detected:
    - Inform user: "Existing codebase detected. Running codebase analysis..."
    - Use the Agent tool to spawn codebase synthesizer:
      - Load role instructions from `src/modules/roles/role-codebase-synthesizer.md` (read the file first)
      - Pass project directory path and any scan manifest data
      - Agent writes CODEBASE-ANALYSIS.md to .planning/research/
    - Wait for completion before proceeding

    If greenfield:
    - Skip codebase synthesizer
    - Note to research agents that this is a greenfield project

    **Step 7: Parallel Research Agents**
    First, ensure .planning/research/ directory exists (already done in Step 5).

    Spawn ALL 5 research agents in parallel using the Agent tool:
    - Read each role file from src/modules/roles/ before spawning
    - Each agent receives: project name, description, model selection, and brownfield analysis (CODEBASE-ANALYSIS.md content if available)
    - Each agent writes its output to .planning/research/{STACK,FEATURES,ARCHITECTURE,PITFALLS,OVERSIGHTS}.md
    - Instruct each agent to use Context7 MCP for documentation lookups when available

    If parallel spawning fails (Claude Code limitation), fall back to sequential execution with a note to the user.

    Wait for ALL 5 agents to complete.

    **Step 8: Research Synthesis**
    Spawn the research synthesizer agent:
    - Read role instructions from src/modules/roles/role-research-synthesizer.md
    - Pass all 5 research output files as context
    - Agent produces .planning/research/SUMMARY.md

    **Step 9: Roadmap Generation**
    Spawn the roadmapper agent:
    - Read role instructions from src/modules/roles/role-roadmapper.md
    - Pass: SUMMARY.md content, project description, team size, model selection
    - Agent returns structured JSON with roadmap content, STATE.json structure, and contracts

    Present roadmap summary to user. Use AskUserQuestion:
    - "Accept roadmap" -- Proceed with this roadmap
    - "Request changes" -- Describe what to change (re-invoke roadmapper with feedback)
    - "Cancel" -- Exit without roadmap

    If accepted:
    - Write ROADMAP.md using the roadmapper's output
    - Write CONTRACT.json files to .planning/sets/{set-name}/CONTRACT.json for each set
    - Update STATE.json with milestone/set/wave/job structure using state-machine.cjs via CLI

    **Step 10: Completion**
    Display summary:
    - Project name, model selection, team size
    - Number of sets, waves, jobs planned
    - Next steps: "Run /rapid:context to generate project context files"

    **Error handling:** At each agent step, if the agent fails or returns an error, use AskUserQuestion to offer Retry/Skip/Cancel (per v1.1 ERRR-03 pattern). Do NOT use bare STOP.

    **Important constraints:**
    - Agents must NOT write STATE.json directly -- SKILL.md uses CLI commands for atomic writes
    - All 5 research agents are independent -- no agent reads another research agent's output
    - Contracts are generated together by the roadmapper (unified), not independently per set
