    ## Detection Pipeline (MERG-01)

    - detectTextualConflicts(cwd, setBranch, baseBranch): Attempts git merge --no-commit --no-ff, parses CONFLICT output, always runs git merge --abort in finally block. Returns {hasConflicts, conflicts: [{file, type, detail}]}
    - detectStructuralConflicts(cwd, setBranch, baseBranch): Gets changed files from both branch and base since branch point, finds overlapping files, extracts function/class names from diff hunks via regex, reports overlapping function modifications. Returns {conflicts: [{file, type: 'structural', functions: string[]}]}
    - detectDependencyConflicts(cwd, setBranch, baseBranch): For overlapping changed files, extracts require/import statements from branch and base versions, reports added/removed dependencies. Returns {conflicts: [{file, type: 'dependency', added, removed}]}
    - detectAPIConflicts(cwd, setBranch, baseBranch): For overlapping changed files, extracts module.exports/export statements from branch and base, reports added/removed/changed exports. Returns {conflicts: [{file, type: 'api', exports: string[], detail}]}
    - detectConflicts(cwd, setName, baseBranch): Orchestrates L1-L4, returns structured report with all levels. L5 semantic is null (filled by merger agent via Plan 03)
    - Helper: getChangedFiles(cwd, branch, baseBranch) using git diff --name-only
    - Helper: getFileContent(cwd, ref, filePath) using git show ref:filePath
    - Helper: getDiffHunks(cwd, branch1, branch2, file) using git diff
    - Helper: extractFunctionNames(diffContent) via regex
    - Helper: extractDependencies(content) via require/import regex
    - Helper: extractExports(content) via module.exports/export regex
    - Helper: parseConflictFiles(gitOutput) extracts filenames from CONFLICT lines
    - Helper: getBranchPoint(cwd, branch, baseBranch) using git merge-base

    Test cases:
    - detectTextualConflicts returns hasConflicts=false when no conflicts exist
    - detectTextualConflicts returns hasConflicts=true with file list when conflicts exist
    - detectTextualConflicts always aborts merge (working tree clean after detection)
    - detectStructuralConflicts returns empty when no overlapping function modifications
    - detectStructuralConflicts detects when same function is modified in both branches
    - detectDependencyConflicts detects added/removed dependencies in overlapping files
    - detectAPIConflicts detects changed exports in overlapping files
    - extractFunctionNames extracts function declarations, const/let arrow functions
    - extractDependencies extracts both require() and import-from patterns
    - extractExports extracts module.exports and named exports
    - parseConflictFiles extracts file paths from git conflict output
    - detectConflicts orchestrates all L1-L4 and returns structured report with L5=null

    ## Resolution Cascade (MERG-02)

    - tryDeterministicResolve(conflict): Tier 1 -- auto-resolves non-overlapping changes, whitespace-only conflicts, formatting differences. Returns {resolved: boolean, confidence: 1.0, resolution: string}
    - tryHeuristicResolve(conflict, ownership, dagOrder): Tier 2 -- uses OWNERSHIP.json to prefer file owner's version, DAG order to prefer earlier-wave set changes, common patterns (both added to array -> merge both, both added imports -> dedup merge). Returns {resolved: boolean, confidence: 0.7-0.9, resolution: string, signal: string}
    - resolveConflicts(detectionResults, options): Iterates through all conflicts, tries T1 then T2, marks remaining as needsAgent for T3. Returns [{conflict, tier, resolved, confidence, resolution?, needsAgent?}]

    Test cases:
    - tryDeterministicResolve resolves non-overlapping addition (confidence=1.0)
    - tryDeterministicResolve returns unresolved for overlapping changes
    - tryHeuristicResolve prefers file owner's version when ownership available
    - tryHeuristicResolve prefers earlier-wave set when DAG order available
    - tryHeuristicResolve merges both entries for array-addition pattern
    - resolveConflicts cascades through tiers correctly
    - resolveConflicts marks unresolvable conflicts as needsAgent
