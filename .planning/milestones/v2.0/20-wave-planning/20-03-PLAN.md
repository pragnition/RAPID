<task type="auto">
  <name>Task 1: /rapid:wave-plan skill</name>
  <files>skills/wave-plan/SKILL.md</files>
  <action>
    Create `skills/wave-plan/SKILL.md` that orchestrates the full research-plan-validate pipeline for a wave. This skill spawns agents sequentially and validates the output.

    **Frontmatter:**
    ```
    ---
    name: wave-plan
    description: Run the wave planning pipeline -- research, wave plan, job plans, and contract validation
    allowed-tools: Bash(rapid-tools:*), Agent, AskUserQuestion, Read, Write, Glob, Grep
    ---
    ```

    **Steps:**

    **Step 1: Environment Setup** -- Standard RAPID_TOOLS env loading preamble (identical to set-init and discuss skills).

    **Step 2: Resolve Wave and Validate State** -- Accept wave ID (e.g., `/rapid:wave-plan wave-1` or `/rapid:wave-plan <set-id> <wave-id>`).
    - Run `node "${RAPID_TOOLS}" wave-plan resolve-wave <waveId>` to find the wave
    - If ambiguous, use AskUserQuestion to pick the set
    - Verify wave status is 'discussing' (has been discussed). If status is:
      - 'pending': "Wave has not been discussed yet. Run /rapid:discuss first." then STOP
      - 'planning' or later: AskUserQuestion: "Wave already has plans. Re-plan from scratch? / View existing plans / Cancel"
    - Verify WAVE-CONTEXT.md exists at `.planning/waves/{setId}/{waveId}/WAVE-CONTEXT.md`. If missing: "Discussion context missing. Run /rapid:discuss first." then STOP.
    - Transition wave to planning: `node "${RAPID_TOOLS}" state transition wave {milestoneId} {setId} {waveId} planning`

    **Step 3: Spawn Wave Research Agent**
    Display progress: "Researching implementation specifics for wave {waveId}..."

    Read context to provide to the agent:
    - Read `.planning/waves/{setId}/{waveId}/WAVE-CONTEXT.md`
    - Read `.planning/sets/{setId}/CONTRACT.json`
    - Read `.planning/sets/{setId}/SET-OVERVIEW.md` (if exists)
    - Read targeted source files referenced in CONTRACT.json exports (file field)

    Use the Agent tool to spawn the wave-researcher:
    ```
    Agent tool call:
    - Prompt: role-wave-researcher instructions + all context above
    - Task: "Research implementation specifics for wave '{waveId}' in set '{setId}'. Write WAVE-RESEARCH.md."
    - Context: WAVE-CONTEXT.md content, CONTRACT.json content, SET-OVERVIEW.md content, source file contents
    - Instruction: "Use Context7 MCP for documentation lookups. Write output to .planning/waves/{setId}/{waveId}/WAVE-RESEARCH.md"
    ```

    After agent completes:
    - Verify WAVE-RESEARCH.md was created
    - Display brief summary (first 5 lines)
    - If agent failed (BLOCKED return), display error and use AskUserQuestion: "Research agent failed: {reason}. Retry / Skip research / Cancel"

    **Step 4: Spawn Wave Planner Agent**
    Display progress: "Creating wave plan for {waveId}..."

    Read additional context:
    - Read `.planning/waves/{setId}/{waveId}/WAVE-RESEARCH.md`
    - Read `.planning/sets/OWNERSHIP.json` (if exists)
    - Get wave jobs from STATE.json: `node "${RAPID_TOOLS}" state get wave {milestoneId} {setId} {waveId}`

    Use the Agent tool to spawn the wave-planner:
    ```
    Agent tool call:
    - Prompt: role-wave-planner instructions + all context
    - Task: "Produce WAVE-PLAN.md for wave '{waveId}' with per-job summaries, file assignments, and coordination notes."
    - Context: WAVE-CONTEXT.md, WAVE-RESEARCH.md, CONTRACT.json, SET-OVERVIEW.md, OWNERSHIP.json, wave jobs from STATE.json
    - Instruction: "Write output to .planning/waves/{setId}/{waveId}/WAVE-PLAN.md"
    ```

    After agent completes:
    - Verify WAVE-PLAN.md was created
    - Display job count and complexity breakdown
    - If agent failed, display error and use AskUserQuestion: "Wave planner failed: {reason}. Retry / Cancel"

    **Step 5: Spawn Job Planner Agents (One Per Job)**
    Display progress: "Creating detailed plans for {N} jobs..."

    Read WAVE-PLAN.md to extract job IDs and their summaries.

    For each job in the wave:
    - Read the job's section from WAVE-PLAN.md
    - Read targeted source files for this specific job (from WAVE-PLAN.md key files)

    Spawn job planner agents. If there are 3+ jobs, spawn them in parallel using multiple Agent tool calls. If 1-2 jobs, spawn sequentially.

    ```
    Agent tool call (per job):
    - Prompt: role-job-planner instructions + job-specific context
    - Task: "Produce {jobId}-PLAN.md for job '{jobId}' with detailed implementation steps."
    - Context: WAVE-PLAN.md (full, with this job's section highlighted), WAVE-RESEARCH.md, WAVE-CONTEXT.md, CONTRACT.json, source files for this job
    - Instruction: "Write output to .planning/waves/{setId}/{waveId}/{jobId}-PLAN.md"
    ```

    After all agents complete:
    - List which JOB-PLAN.md files were created
    - If any agent failed, list failures and use AskUserQuestion: "Some job plans failed. Retry failed / Continue without them / Cancel"

    **Step 6: Contract Validation Gate**
    Display progress: "Validating job plans against contracts..."

    Run the contract validation via CLI:
    ```bash
    node "${RAPID_TOOLS}" wave-plan validate-contracts {setId} {waveId}
    ```

    Parse the JSON output (result, violations, autoFixes):

    - If result is "PASS": "All job plans validated successfully against contracts."
    - If result is "PASS_WITH_WARNINGS": Display auto-fixes applied and minor violations noted. Continue.
    - If result is "FAIL": Display major violations. For each major violation, use AskUserQuestion:
      ```
      "Contract violation: {violation detail}"
      Options:
      - "Fix plan" -- "Remove the problematic import/export and find alternative"
      - "Update contract" -- "Add the missing export to the source set's contract"
      - "Override" -- "Proceed anyway -- may be resolved during execution"
      ```

    Write VALIDATION-REPORT.md to `.planning/sets/{setId}/VALIDATION-REPORT.md` with full results.

    **Step 7: Commit and Present Results**
    - Commit all planning artifacts:
    ```bash
    git add .planning/waves/{setId}/{waveId}/ .planning/sets/{setId}/VALIDATION-REPORT.md
    git commit -m "plan({setId}): wave {waveId} planning complete -- research, wave plan, {N} job plans, validation"
    ```

    - Display summary:
      - Wave plan: {N} jobs with complexity breakdown
      - Job plans: {N} created (list each with file path)
      - Validation: {PASS/PASS_WITH_WARNINGS/FAIL}
      - Auto-fixes applied: {N}
      - Violations found: {N major}, {N minor}

    - Use AskUserQuestion for next steps:
      - "Run /rapid:execute to start executing this wave" -- proceed to execution
      - "Plan another wave" -- run /rapid:wave-plan for a different wave
      - "View job plans" -- display the JOB-PLAN.md files
      - "View /rapid:status" -- see current project state

    **Important patterns:**
    - Use tilde ~ for node paths in shell commands per CLAUDE.md
    - Sequential agent spawning for research -> wave-plan (each depends on prior output)
    - Parallel agent spawning for job planners (independent, share read-only inputs)
    - All state transitions via `node "${RAPID_TOOLS}" state transition` CLI (never edit STATE.json directly)
    - If any agent returns BLOCKED, pipeline stops with user options
    - The skill should be ~180-250 lines
