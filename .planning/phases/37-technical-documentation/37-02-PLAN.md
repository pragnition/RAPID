**Part B: Create docs/troubleshooting.md**

Use the 6 failure modes from RESEARCH.md. Each uses the symptom/cause/fix card format.

**Structure:**

1. **Introduction** (1-2 sentences): Common issues and their resolutions.

2. **6 Troubleshooting Cards** (from RESEARCH.md):

   1. **RAPID_TOOLS not set**
      - Symptom: `[RAPID ERROR] RAPID_TOOLS is not set` on any command
      - Cause: Shell config not sourced after install, or .env missing
      - Fix: Run `/rapid:install` or manually set in shell config and .env

   2. **Stale lock files**
      - Symptom: Commands hang or `[RAPID] Lock "state-machine" compromised`
      - Cause: Agent crashed mid-operation
      - Fix: Auto-expire after 5 minutes. Manual: delete `.planning/.locks/*.target.lock`
      - Cross-reference: [State Machines](state-machines.md)

   3. **STATE.json corruption**
      - Symptom: `Cannot transition: STATE.json is missing or invalid`
      - Cause: Concurrent writes, manual edits, crash during write
      - Fix: `node "${RAPID_TOOLS}" state detect-corruption` then `node "${RAPID_TOOLS}" state recover`
      - Cross-reference: [State Machines](state-machines.md)

   4. **Worktree cleanup blocked**
      - Symptom: `/rapid:cleanup` shows `removed: false, reason: "dirty"`
      - Cause: Uncommitted changes in worktree
      - Fix: Commit/stash changes then retry, or `git worktree remove --force <path>` (destructive)

   5. **Subagent timeout / missing return marker**
      - Symptom: `Warning: Job '{jobId}' returned without a RAPID:RETURN marker`
      - Cause: Context window limit, crash, or failed return emission
      - Fix: Re-run `/rapid:execute <set-id>` -- smart re-entry handles stale executing states

   6. **Merge conflicts during merge pipeline**
      - Symptom: `git merge-tree --write-tree` returns exit code 1
      - Cause: Two sets modified overlapping files
      - Fix: Follow merge pipeline prompts. High-confidence auto-resolves; low-confidence escalates to human.

Each card uses the exact format from RESEARCH.md patterns section. Cross-reference state machine docs for state-related issues (cards 2 and 3).

**Tone:** Direct and practical. Users reading this are troubleshooting a problem -- get to the fix fast.
