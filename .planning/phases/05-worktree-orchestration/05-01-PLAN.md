  <action>
    Create `rapid/src/lib/worktree.test.cjs` first with failing tests, then implement `rapid/src/lib/worktree.cjs`.

    **worktree.cjs implementation details:**

    Follow 'use strict' + CommonJS pattern (`module.exports = { ... }`).

    `gitExec(args, cwd)`:
    - Use `child_process.execSync` with `encoding: 'utf-8'`, `stdio: ['pipe', 'pipe', 'pipe']`, `timeout: 30000`
    - Wrap in try/catch; return `{ ok: true, stdout: result.trim() }` or `{ ok: false, exitCode: err.status, stderr: (err.stderr || '').toString().trim() }`

    `detectMainBranch(cwd)`:
    - Run `git rev-parse --abbrev-ref HEAD` via gitExec
    - If result is 'HEAD' (detached), throw Error('Detached HEAD -- cannot determine base branch')
    - Return the branch name string

    `createWorktree(projectRoot, setName)`:
    - Branch name: `rapid/${setName}`
    - Worktree path: `path.resolve(projectRoot, '.rapid-worktrees', setName)`
    - Ensure `.rapid-worktrees/` dir exists (mkdir -p)
    - Run `git worktree add -b ${branch} "${worktreePath}" HEAD` via gitExec from projectRoot
    - On failure, check stderr for 'already exists' and throw specific error messages
    - Return `{ branch, path: worktreePath }`

    `removeWorktree(projectRoot, worktreePath)`:
    - Run `git worktree remove "${worktreePath}"` via gitExec from projectRoot
    - On failure with stderr containing 'modified or untracked' or exit code 128: return `{ removed: false, reason: 'dirty', message: 'Worktree has uncommitted changes' }`
    - On other failure: return `{ removed: false, reason: 'error', message: result.stderr }`
    - On success: return `{ removed: true }`

    `listWorktrees(projectRoot)`:
    - Run `git worktree list --porcelain` via gitExec
    - Parse blocks separated by `\n\n`; for each block parse lines: `worktree ` -> path, `HEAD ` -> head, `branch refs/heads/` -> branch, `detached` -> detached:true, `locked` -> locked:true
    - Return array of entry objects

    `loadRegistry(cwd)`:
    - Read `.planning/worktrees/REGISTRY.json` via fs.readFileSync
    - If file doesn't exist, return `{ version: 1, worktrees: {} }`
    - Parse and return

    `writeRegistry(cwd, registry)`:
    - Ensure `.planning/worktrees/` directory exists (fs.mkdirSync recursive)
    - Write JSON.stringify(registry, null, 2) + '\n' to `.planning/worktrees/REGISTRY.json`

    `registryUpdate(cwd, updateFn)`:
    - Acquire lock via `acquireLock(cwd, 'worktree-registry')` from lock.cjs
    - Load registry, call updateFn(registry), write result, release lock in finally block
    - Return the updated registry

    `reconcileRegistry(cwd)`:
    - Load registry and call listWorktrees to get actual git state
    - For each registry entry: if no matching worktree in git list, mark entry.status = 'orphaned'
    - For each git worktree with branch starting with 'rapid/': if not in registry, add entry with setName derived from branch (strip 'rapid/' prefix), phase: 'Created', status: 'active'
    - Write reconciled registry and return it

    `ensureWorktreeDir(projectRoot)`:
    - `fs.mkdirSync(path.join(projectRoot, '.rapid-worktrees'), { recursive: true })`

    **Test setup:** Each test creates a real git repo in `os.tmpdir()` via `git init`, adds an initial commit, and cleans up after. Use `node:test` describe/it/beforeEach/afterEach pattern. Use `node:assert/strict`.

    For tests that need worktree operations, the temp repo needs at least one commit (worktrees require a non-empty repo). Create with:
    ```
    execSync('git init', { cwd: tmpDir });
    execSync('git commit --allow-empty -m "init"', { cwd: tmpDir });
    ```

    For registry tests, create a `.planning/` directory in the temp dir.

    Add `worktree` to the USAGE string:
    ```
    worktree create <set-name>  Create worktree and branch for a set
    worktree list               List all registered worktrees with status
    worktree cleanup <set-name> Remove a worktree (blocks if dirty)
    worktree reconcile          Sync registry with actual git state
    ```

    Implement `async function handleWorktree(cwd, subcommand, args)`:

    **create subcommand:**
    - Require setName from args[0]
    - Call `worktree.createWorktree(cwd, setName)`
    - On success, call `worktree.registryUpdate(cwd, (reg) => { reg.worktrees[setName] = { setName, branch: 'rapid/' + setName, path: relative-path, phase: 'Created', status: 'active', wave: null, createdAt: new Date().toISOString() }; return reg; })`
    - Output JSON `{ created: true, branch, path, setName }`
    - On error, output JSON `{ created: false, error: err.message }` and exit 1

    **list subcommand:**
    - Call `worktree.reconcileRegistry(cwd)` first to ensure freshness
    - Output JSON `{ worktrees: Object.values(registry.worktrees) }`

    **cleanup subcommand:**
    - Require setName from args[0]
    - Load registry, find entry for setName
    - If entry not found, error and exit 1
    - Resolve absolute path from entry's relative path
    - Call `worktree.removeWorktree(cwd, absolutePath)`
    - If removed: deregister from REGISTRY.json via registryUpdate (delete entry), output `{ removed: true, setName }`
    - If not removed (dirty): output `{ removed: false, reason, setName, message }` and exit 1

    **reconcile subcommand:**
    - Call `worktree.reconcileRegistry(cwd)`

    - Output `{ reconciled: true, orphaned: N, discovered: N }`
