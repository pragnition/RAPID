<task type="auto">
  <name>Task 2: Rewrite execute-set SKILL.md for v3</name>
  <files>skills/execute-set/SKILL.md</files>
  <action>
    Write skills/execute-set/SKILL.md FROM SCRATCH (do not modify the existing 516-line v2 file). The v3 execute-set follows the established v3 skill pattern from Phase 43 skills.

    **Frontmatter:**
    ```yaml
    ---
    description: Execute all waves in a set -- one executor per wave, sequential, artifact-based re-entry
    allowed-tools: Bash(rapid-tools:*), Agent, AskUserQuestion, Read, Write, Glob, Grep
    ---
    ```

    **Structure (follow v3 skill pattern from plan-set/discuss-set):**

    **Step 0: Environment + Banner**
    - Standard env preamble (RAPID_ROOT, .env loading, RAPID_TOOLS check)
    - `node "${RAPID_TOOLS}" display banner execute-set`

    **Step 1: Resolve Set and Validate Preconditions**
    - Parse set-id argument (numeric or string). Use `resolve set` CLI.
    - If no set-id provided, use AskUserQuestion to ask which set.
    - Load state: `node "${RAPID_TOOLS}" state get --all`
    - **Status check:**
      - If `planning`: Expected state. Continue.
      - If `executing`: Re-entry scenario. Continue (will pick up from last complete wave).
      - If `pending` or `discussing`: Error -- suggest discuss-set or plan-set first. Show breadcrumb. STOP.
      - If `complete` or `merged`: Already done. Suggest /rapid:review. STOP.
    - Check for `--gaps` flag: if present, only execute waves without WAVE-COMPLETE.md markers (gap closure mode).
    - Read PLAN.md files: Glob `.planning/sets/${SET_ID}/wave-*-PLAN.md`. If none found, suggest plan-set. STOP.
    - Count total waves from PLAN.md files.
    - Determine worktree path from `.rapid-worktrees/{SET_ID}` or equivalent.

    **Step 2: Re-Entry Detection (Artifact-Based)**
    - For each wave N (1 to total_waves):
      - Check if `.planning/sets/${SET_ID}/WAVE-${N}-COMPLETE.md` exists
      - If marker exists: read it, extract commit hashes, verify commits exist on worktree branch via `git -C {worktreePath} log --oneline main..HEAD | grep {commitHash}`
      - If marker exists AND commits verified: mark wave as complete (skip)
      - If marker missing OR commits don't match: mark wave for execution
    - Display re-entry summary:
      ```
      Wave 1: complete (skipping)
      Wave 2: pending (will execute)
      Wave 3: pending (will execute)
      ```
    - If ALL waves complete: "All waves in set '{SET_ID}' already complete." Suggest /rapid:review. STOP.
    - If not first run (some waves complete): "Resuming execution from wave {N}..."

    **Step 3: State Transition**
    - If set is in `planning` state: `node "${RAPID_TOOLS}" state transition set "${MILESTONE}" "${SET_ID}" executing`
    - If set is already in `executing` state: skip transition (re-entry).

    **Step 4: Execute Waves Sequentially**
    - For each wave N that needs execution:

      **4a: Display progress**
      ```
      Executing wave {N} of {total_waves} for set '{SET_ID}'...
      ```

      **4b: Spawn rapid-executor agent**
      Spawn the **rapid-executor** agent with this task:
      ```
      Implement wave {N} for set '{SET_ID}'.

      ## Your PLAN
      {Full content of wave-{N}-PLAN.md}

      ## Commit Convention
      After each task, commit with: type({SET_ID}): description
      Where type is feat|fix|refactor|test|docs|chore

      ## Working Directory
      {worktreePath}
      ```

      **4c: Process RAPID:RETURN**

      **If COMPLETE:**
      1. Verify at least one commit exists: `git -C {worktreePath} log --oneline main..HEAD` (must have commits since execution start)
      2. Check no uncommitted changes: `git -C {worktreePath} status --porcelain` (should be empty or only untracked)
      3. If both pass: Write WAVE-{N}-COMPLETE.md marker to `.planning/sets/${SET_ID}/WAVE-{N}-COMPLETE.md`:
         ```markdown
         # Wave {N} Complete

         **Set:** {SET_ID}
         **Wave:** {N}
         **Completed:** {ISO timestamp}
         **Executor commits:** {comma-separated commit hashes from git log}
         **Branch:** rapid/{SET_ID}
         **Reconciliation:** basic-pass
         ```
      4. Display: "Wave {N}: complete."
      5. Continue to next wave.

      **If CHECKPOINT:**
      1. Write handoff file to `.planning/sets/${SET_ID}/WAVE-{N}-HANDOFF.md` with checkpoint data
      2. Display: "Wave {N}: paused at checkpoint. Resume with /rapid:execute-set {SET_INDEX}"
      3. STOP.

      **If BLOCKED:**
      1. Display blocker: "Wave {N}: BLOCKED ({category}: {detail})"
      2. STOP.

      **If no RAPID:RETURN found:**
      1. Display: "Warning: Wave {N} executor returned without RAPID:RETURN marker."
      2. Check if executor made commits anyway (git log). If commits exist, write marker and continue. If no commits, STOP with error.

    **Step 5: Post-Execution Verification**
    - After ALL waves complete:
    - Read success criteria from ROADMAP.md for this set (parse the set description and success criteria section).
    - Spawn the **rapid-verifier** agent:
      ```
      Verify execution of set '{SET_ID}'.

      ## Success Criteria (from ROADMAP.md)
      {success criteria for this set}

      ## Wave Plans and Objectives
      {For each wave-N-PLAN.md: wave N objective summary}

      ## Working Directory
      {worktreePath}

      ## Instructions
      Check each success criterion against the actual implementation.
      For each criterion: verify the stated behavior works (run tests, check files exist, verify connections).
      Return COMPLETE with a `gaps` array listing any unmet criteria (empty if all pass).
      ```
    - Parse RAPID:RETURN from verifier.
    - If `gaps` array is empty: All criteria met. Continue to Step 6.
    - If `gaps` array has items:
      1. Write GAPS.md to `.planning/sets/${SET_ID}/GAPS.md` listing unmet criteria
      2. Display: "{N} gaps found. See GAPS.md."
      3. Display: "To close gaps: /rapid:plan-set {SET_INDEX} --gaps then /rapid:execute-set {SET_INDEX} --gaps"
      4. Continue to Step 6 (verification is non-blocking per locked decision).

    **Step 6: Complete Set**
    - Transition set to complete: `node "${RAPID_TOOLS}" state transition set "${MILESTONE}" "${SET_ID}" complete`
    - Commit marker files and GAPS.md (if any):
      ```bash
      git add ".planning/sets/${SET_ID}/WAVE-*-COMPLETE.md"
      git add ".planning/sets/${SET_ID}/GAPS.md" 2>/dev/null || true
      git commit -m "execute-set(${SET_ID}): complete execution"
      ```
    - Display final summary:
      ```
      Set '{SET_ID}' execution complete.
      Waves: {completed}/{total}
      ```
    - Display next step: `> **Next step:** /rapid:review {SET_INDEX}`
    - Display progress breadcrumb:
      ```
      init [done] > start-set [done] > discuss-set [done] > plan-set [done] > execute-set [done] > review > merge
      ```

    **Error Handling section:**
    - RAPID_TOOLS not set: suggest /rapid:install
    - STATE.json missing: suggest /rapid:init
    - No PLAN.md files: suggest /rapid:plan-set
    - Set in wrong state: show breadcrumb with failure point

    **Anti-Patterns section (CRITICAL -- must be in the SKILL.md):**
    - Do NOT reference JOB-PLAN.md -- v3 uses per-wave PLAN.md only
    - Do NOT use `state transition wave` or `state transition job` -- v3 has set-level state only
    - Do NOT detect agent teams mode -- v3 uses subagents only
    - Do NOT use `execute detect-mode`, `execute job-status`, `execute reconcile-jobs`, `wave-plan list-jobs`
    - Do NOT run per-wave reconciliation reports or lean review per wave
    - Do NOT prompt between waves on success -- auto-advance. Only CHECKPOINT or BLOCKED stops.
    - Do NOT reference any v2 commands listed in the deprecated section of 44-RESEARCH.md

    **Key Principles section:**
    - One executor per wave, sequential waves
    - Artifact-based re-entry: WAVE-COMPLETE.md + git commit verification
    - RAPID:RETURN protocol for structured executor returns
    - Subagents only -- no agent teams
    - Set-level state only: planning -> executing -> complete
    - Non-blocking verification: GAPS.md report, user decides next action
    - Progress breadcrumbs on completion and errors
