Then create src/lib/review.cjs with:

1. **Zod schemas** (following state-schemas.cjs pattern with Zod 3.24.4):
   - `ReviewIssue`: id (string), type (enum: artifact/static/contract/test/bug/uat), severity (enum: critical/high/medium/low), file (string), line (number optional), description (string), autoFixAttempted (boolean default false), autoFixSucceeded (boolean default false), source (enum: lean-review/unit-test/bug-hunt/uat), status (enum: open/fixed/deferred/dismissed, default 'open'), createdAt (string), fixedAt (string optional)
   - `ReviewIssues`: waveId (string), setId (string), issues (array of ReviewIssue), lastUpdatedAt (string)

2. **Scoping functions**:
   - `scopeWaveForReview(cwd, worktreePath, baseBranch)`: Uses execute.getChangedFiles() to get changed files, then findDependents() for one-hop dependents. Returns { changedFiles, dependentFiles, totalFiles }.
   - `findDependents(cwd, changedFiles)`: Searches project files for import/require statements referencing any of the changed files. Uses fs.readdirSync recursively (skip node_modules, .git, .planning). Returns unique dependent file paths not already in changedFiles.

3. **Issue management**:
   - `logIssue(cwd, setId, waveId, issue)`: Validates issue with ReviewIssue.parse(), reads/creates REVIEW-ISSUES.json at `.planning/waves/{setId}/{waveId}/REVIEW-ISSUES.json`, appends issue, writes atomically.
   - `loadSetIssues(cwd, setId)`: Reads all REVIEW-ISSUES.json files across all wave directories for the set. Returns flat array with waveId attached to each issue.
   - `updateIssueStatus(cwd, setId, waveId, issueId, newStatus)`: Finds issue by id in the wave's REVIEW-ISSUES.json, updates status field. If newStatus is 'fixed', sets fixedAt to current ISO timestamp.

4. **Constants**:
   - `REVIEW_CONSTANTS`: { MAX_BUGFIX_CYCLES: 3, ISSUE_TYPES: ['artifact', 'static', 'contract', 'test', 'bug', 'uat'], SEVERITY_LEVELS: ['critical', 'high', 'medium', 'low'] }

5. **Summary generation**:
   - `generateReviewSummary(setId, issues)`: Produces markdown string for REVIEW-SUMMARY.md with sections: overview (total issues, by severity), per-wave breakdown, issues by status (open/fixed/deferred/dismissed), deferred count warning if >5.

Use `'use strict';` at top. CommonJS module.exports. Follow existing lib module patterns (require('fs'), require('path'), require('zod')).
