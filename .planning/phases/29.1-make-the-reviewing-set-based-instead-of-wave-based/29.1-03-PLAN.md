    #### Step 4a: Unit Test Stage (if selected)
    - **If chunks.length <= 1 (single chunk or no chunking):**

      - Follow same plan-approve-execute flow as before (3a.2 through 3a.5)

      - Agent context: "Review set '{setId}', chunk: {chunk.dir} ({chunk.files.length} files)"
      - Collect all test plans from all chunks
      - Present combined test plan for approval (grouped by chunk/directory as described in RESEARCH.md)
      - After approval, spawn execution agents per chunk in parallel
      - Merge results: sum testsWritten/testsPassed/testsFailed across chunks

    Write REVIEW-UNIT.md to `.planning/waves/{setId}/REVIEW-UNIT.md` (NOT wave subdirectory).
    Update report template: replace "Wave {waveId}" with "Set {setId}".

    Log test failures:

    Note: look up originatingWave from waveAttribution map for the test's target file.

    #### Step 4b: Bug Hunt Stage (if selected)
    - **Cycle loop (max 3, same as before):**
    - **Cycle 1 scope:** Use full set scope (or chunks if chunked)
    - **Cycle 2-3 scope:** Flat scope on bugfix-modified files only, NO re-chunking

    - **Cycle 1 with chunks.length > 1:**
      1. Spawn rapid-bug-hunter agents PER CHUNK in parallel (up to 5 concurrent)
      2. Each hunter gets: "Analyze set '{setId}', chunk: {chunk.dir}"
      3. Collect all findings from all hunters
      4. Prefix chunk index to finding IDs to prevent collisions: C1-F-001, C2-F-001
      5. Merge all findings into single array
      6. Spawn ONE rapid-devils-advocate on merged findings
      7. Spawn ONE rapid-judge on merged findings + advocate assessments
      8. Handle DEFERRED rulings (same AskUserQuestion flow as before)
      9. Write REVIEW-BUGS.md to `.planning/waves/{setId}/REVIEW-BUGS.md`
      10. Spawn rapid-bugfix on accepted bugs
      11. Record modifiedFiles for next cycle

    - **Cycle 1 with single chunk:** Same as above but one hunter, no chunk prefixing

    - **Cycles 2-3:** Flat scope on modifiedFiles, single hunter (no chunking), then advocate + judge
