    **1. Create src/modules/roles/role-merger.md:**

    Write the merger agent role prompt following the established pattern from other roles (role-reviewer.md, role-bugfix.md). The prompt must include:

    **Header section:**
    - Title: RAPID Merger Agent
    - Description: Performs semantic conflict detection (Level 5) and AI-assisted conflict resolution (Tier 3) for the merge pipeline

    **Context section (placeholders for skill injection):**
    - `{SET_NAME}` -- the set being merged
    - `{BASE_BRANCH}` -- the target branch (main/master)
    - `{SET_CONTEXT}` -- the merging set's CONTEXT.md/plans
    - `{OTHER_SET_CONTEXTS}` -- contexts of already-merged sets in the same wave
    - `{DETECTION_REPORT}` -- structured L1-L4 conflict detection results
    - `{CONTRACTS}` -- interface contracts for the merging set and any related sets
    - `{UNRESOLVED_CONFLICTS}` -- conflicts not resolved by T1/T2 cascade

    **Semantic Detection section (Task 1):**
    - Read both the merging set's CONTEXT.md/plans AND the contexts of sets already merged in this wave
    - Identify INTENT DIVERGENCE: two sets modify related functionality in incompatible ways (e.g., set A changes auth flow, set B changes auth middleware -- changes conflict semantically even if no textual conflict)
    - Identify CONTRACT BEHAVIORAL MISMATCH: merged code violates interface contracts -- not just schema but behavioral expectations
    - Look at function signatures, return values, error handling, side effects
    - For each semantic conflict found, assign a confidence score (0.0-1.0)

    **Resolution section (Task 2):**
    - For each unresolved conflict from the detection report, write resolved code
    - Apply resolution directly to the file in the worktree
    - For each resolution, assign a confidence score:
      - 0.9+: Trivially correct resolution (e.g., both added same import)
      - 0.7-0.9: Clearly correct but requires understanding context
      - 0.5-0.7: Reasonable resolution but alternative interpretations exist
      - <0.5: Low confidence -- should escalate to human
    - If confidence < 0.7 (configurable threshold), do NOT apply -- mark for human escalation

    **Output section:**
    - Emit RAPID:RETURN with status COMPLETE containing:
      - semantic_conflicts: [{description, sets, confidence, file?}]
      - resolutions: [{file, original_conflict, resolution_summary, confidence, applied: boolean}]
      - escalations: [{file, conflict_description, reason, confidence}]
      - all_resolved: boolean (true if no escalations needed)

    **Rules section:**
    - NEVER use git add -A or git add .
    - Stage specific resolved files only
    - Do NOT commit -- the skill orchestrator handles commits
    - Do NOT modify files unrelated to conflicts
    - If a resolution would change API signatures or behavior, escalate rather than apply
