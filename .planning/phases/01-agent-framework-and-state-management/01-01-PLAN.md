    - rapid-tools.cjs: `node rapid-tools.cjs lock acquire state` acquires and prints confirmation

  <action>
    1. Create `rapid/package.json` with name "rapid", version "0.1.0", and dependency on `proper-lockfile@^4.1.2`. No build tools needed yet -- pure CommonJS.

    2. Run `cd ~/RAPID/rapid && npm install` to install proper-lockfile.

    3. Create `rapid/.gitignore` with `node_modules/` entry.

    4. Create `rapid/src/lib/core.cjs`:
       - `output(msg)`: Writes `[RAPID] ${msg}` to stdout
       - `error(msg)`: Writes `[RAPID ERROR] ${msg}` to stderr
       - `findProjectRoot(startDir)`: Walks up directory tree looking for `.planning/` directory. Returns the path containing it, or throws if not found. Start from `startDir || process.cwd()`.
       - `loadConfig(projectRoot)`: Reads `rapid/config.json` relative to project root. Returns parsed JSON or sensible defaults `{ agents: {}, lock_timeout_ms: 300000 }` if file doesn't exist.
       - `resolveRapidDir()`: Returns path to the `rapid/` directory (where the tool lives), using `__dirname` relative resolution.
       - Export all functions.

    5. Create `rapid/src/lib/lock.cjs`:
       - Import `proper-lockfile` as `lockfile`.
       - Constants: `LOCKS_DIR = '.planning/.locks'`, `STALE_THRESHOLD = 300000` (5 min), `RETRY_CONFIG = { retries: 10, factor: 2, minTimeout: 100, maxTimeout: 2000, randomize: true }`.
       - `ensureLocksDir(cwd)`: Creates `.planning/.locks/` directory recursively. Writes `.gitignore` with `*\n!.gitignore\n` inside it if not present.
       - `acquireLock(cwd, lockName)`: Ensures locks dir, creates `${lockName}.target` file if needed (containing `{ pid, timestamp }`), calls `lockfile.lock()` with stale threshold, mtime update interval (half of stale), retry config, `realpath: false`, and `onCompromised` handler that logs warning but doesn't crash. Returns the release function.
       - `isLocked(cwd, lockName)`: Synchronous non-blocking check using `lockfile.checkSync()`. Returns boolean.
       - Export: `{ acquireLock, isLocked, ensureLocksDir }`.

    6. Create `rapid/src/bin/rapid-tools.cjs`:
       - Hashbang `#!/usr/bin/env node`
       - Parse `process.argv` for subcommand routing: `lock acquire <name>`, `lock release <name>`, `lock status <name>`, plus a help/usage fallback.
       - For `lock acquire`: call `acquireLock`, print confirmation JSON `{ "acquired": true, "lock": name }`.
       - For `lock status`: call `isLocked`, print `{ "locked": boolean, "lock": name }`.
       - Unknown commands: print usage listing and exit with code 1.
       - State subcommands will be added in Task 2.

    7. Write tests in `rapid/src/lib/core.test.cjs` and `rapid/src/lib/lock.test.cjs` using Node.js built-in `node:test` and `node:assert` (no test framework dependency needed).
       - core tests: output/error format correctly, findProjectRoot works with temp dir containing .planning/, loadConfig returns defaults when no config exists.
       - lock tests: ensureLocksDir creates directory and .gitignore, acquireLock + release cycle works, isLocked returns correct state, stale lock recovery works (create a lock target manually, write stale PID, verify new acquire succeeds).

    IMPORTANT: Use `~` for home directory paths in shell commands, NOT `$HOME`. Use CommonJS (`require`/`module.exports`) throughout -- no ESM.
