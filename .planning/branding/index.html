Design Principles

Surface over shadow

Elevation is expressed by the surface token (bg-0 through bg-4), never by drop shadows or glass blur. The look remains flat and readable across every theme.

Theme-agnostic semantics

Every semantic token (accent, info, warning, error, highlight, orange) is redefined per theme. Components consume --color-* vars and never reference raw hex.

Badges for lifecycle

Set, wave, and job status render as semantic monospace badges with 20% tinted backgrounds. Never describe lifecycle state in prose.

Deliberate feedback

Every long-running action shows an explicit state — skeleton, progress bar, or status badge. No silent optimistic updates. Command palette on ⌘K.
