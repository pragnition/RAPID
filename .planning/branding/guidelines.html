Color Palette

Everforest Dark is the default theme. Tokens are exposed as CSS custom properties in web/frontend/src/styles/global.css and remapped for each theme variant. Reference tokens by semantic name — never hardcode hex values in components.

Status Badges

Lifecycle state is always rendered as a monospace badge with semantic color at 20% background opacity. Never render lifecycle state as plain prose. Consistent across tables, cards, graph nodes, Kanban columns.

pending discussed planned executed complete merged warning error

Do / Don't

DO

Signal elevation with surface tokens

Stack surface-0 → surface-4 to indicate depth. Nested cards go one level higher than their parent.

<div class="bg-surface-1 border border-border rounded-lg p-4">
  <div class="bg-surface-2 rounded p-3">nested</div>
</div>
DON'T

Use drop shadows or blur

Shadows break the flat terminal-dashboard aesthetic and fail against light theme variants.

<div class="shadow-lg backdrop-blur">...</div>
DO

Render state as a semantic badge

<span class="badge badge-planned">planned</span>
DON'T

Describe state in prose

<p>This set is currently in the planned state.</p>
DO

Use domain terminology

Set, Wave, Job, Contract, Worktree, DAG.

DON'T

Substitute generic alternatives

"task", "module", "feature", "sandbox", "schema", "graph".

Focus & Keyboard

  • All interactive elements render a 2px focus ring in --accent with 2px offset.
  • Command palette opens with Cmd/Ctrl + K and returns focus to the trigger on close.
  • Kanban cards are draggable via pointer AND keyboard (arrow keys to move between columns when focused).
  • Theme swap is triggered by button, never on hover — respecting prefers-reduced-motion.

Screen Readers

  • Status badges include aria-label="status: planned".
  • Graph nodes expose set ID + status via aria-label; SVG uses role="img".
  • Progress bars use role="progressbar" with aria-valuenow, aria-valuemin, aria-valuemax.

Voice & Tone

Third-person, formal, declarative. Avoid you, we, I, let's. Never use emojis, marketing language, or filler words. Use RAPID domain terminology consistently.
