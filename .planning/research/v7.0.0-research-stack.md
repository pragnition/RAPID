- **Latest versions (verified April 2026)**:

- **Release cadence**: ~weekly patch releases. Security-relevant bumps happened at 0.2.101 (SDK dep bumps for GHSA-5474-4w2j-mq4c). Expect churn ‚Äî do not exact-pin like the core libs do; use caret range.

## Compatibility Matrix

Critical decision: **the Agent SDK only ships for Node.js and Python**. RAPID's web backend is Python/FastAPI; RAPID's plugin core is Node/CommonJS. Two viable integration shapes:

1. **Python path (align with web backend)** ‚Äî use `claude-agent-sdk` inside the existing FastAPI app. WebSocket endpoints (FastAPI native) stream SDK messages; `canUseTool` callback maps to `AskUserQuestion`-style browser controls. Python 3.12 easily meets the 3.10+ floor. No new runtime added.
2. **Node path (align with RAPID CLI)** ‚Äî stand up a Node sidecar service that imports `@anthropic-ai/claude-agent-sdk`. Requires adding a second backend runtime (Node alongside Python) and an RPC/IPC bridge, or swapping the Python backend for a Node backend (large change). Wins if the SDK's TypeScript typings or new-feature velocity matter more than runtime uniformity.

**Recommendation: Python path**. The Python SDK exposes the same `query()`, `ClaudeSDKClient`, `ClaudeAgentOptions`, `can_use_tool`, custom MCP tools (`create_sdk_mcp_server`, `@tool`), skills (`setting_sources=["project"]`), plugins, sessions, and hooks. No runtime added, no IPC shim, no bifurcated backend. The only asymmetry is that RAPID's CLI plugin is still driven by Claude Code (unchanged); the webUI drives its own SDK sessions.

### Version constraints to watch

- `zod` peer in the TS SDK accepts `^3.24.1 || ^4.0.0` ‚Äî if we ever adopt the TS SDK, RAPID's current `zod 3.25.76` fits without change.
- TypeScript 5.2+ is required only for the V2 preview's `await using` pattern. Current frontend is TS 5.9 ‚Äî satisfied if we ever use it.
- Claude Code CLI is bundled with both SDKs ‚Äî **do NOT also `npm install -g @anthropic-ai/claude-code`**, which risks version mismatch vs. the bundled binary. The SDK picks its bundled CLI by default.
- MCP server processes spawned by the SDK were previously leaking on session end (fixed in 0.2.101+) ‚Äî require >=0.2.101 minimum.
- 11+ concurrent `query()` calls triggered `MaxListenersExceededWarning` before 0.2.101 ‚Äî relevant if multiple kanban cards execute in parallel.

## Tooling Assessment

- **Build tools (web)**: Vite 8.0.1 / TS 5.9 / Tailwind 4.2 ‚Äî fully up to date; no gaps for adding two new tabs (`agents`, `chats`).
- **State mgmt (web)**: Zustand 5.0 + TanStack Query 5.91 already present. For WS event streams, prefer **TanStack Query's streaming/infinite query** or a Zustand slice rather than adding a new library. No new dep needed.
- **WebSocket on backend**: FastAPI's native `WebSocket` + Starlette is sufficient. No Socket.IO dep required. For browser side, native `WebSocket` or a thin hook is adequate; the demo `simple-chatapp` pattern from the Anthropic SDK demos is directly adaptable.
- **CodeMirror**: Already installed (7 lang packs). Useful for rendering agent tool-use payloads and file diffs inside the chat tab ‚Äî no new dep.
- **Test framework**:
  - Node core: `node:test` ‚Äî continues to work for non-SDK code.
  - Python backend: `pytest` 8.x + `pytest-asyncio` 0.25 already in dev deps ‚Äî can mock the SDK cleanly.
  - **Gap**: a pytest plugin exists called `pytest-claude-agent-sdk` (0.1.0) that provides spying and LLM-as-judge assertions. Useful for agent behavioral tests but adds a medium-reputation external dep; consider for integration tests only.
- **CI/CD**: No CI config visible in the snapshot. Adding an agent-integration test tier will need either recorded SDK fixtures or a gated live-key test job ‚Äî a decision for planning, not a stack blocker.
- **Async runtime parity**: Python SDK is `asyncio`-native; FastAPI is too. No event-loop bridging needed inside a single worker.

## Stack Risks

1. **Pre-1.0 SDK churn** ‚Äî Impact: breaking changes possible between minor releases. Mitigation: pin to `>=0.1.59,<0.2.0` (Python) / `>=0.2.109,<0.3.0` (TS if ever used) and add SDK upgrade to the repo's version-pin test pattern that `src/lib/version.test.cjs` enforces for core deps.
2. **Bundled Claude Code CLI drift** ‚Äî Impact: the SDK spawns its bundled CLI, which has its own version; tool and skill behavior can shift when the CLI underneath changes. Mitigation: record the CLI version (`setting_sources=["project"]` + `system init` message's metadata) at session start and surface it in the webUI's agent session log.
3. **API key scope and billing** ‚Äî Impact: each web-driven agent session costs tokens against one `ANTHROPIC_API_KEY`. Mitigation: enforce `maxTurns` and `maxBudgetUsd` on every SDK call; surface live cost tracking in the webUI (the SDK emits cost in result messages).
4. **Concurrency limits** ‚Äî Impact: SDK pre-0.2.101 warned on 11+ concurrent `query()` calls; the kanban feature can plausibly fan out to many concurrent agents. Mitigation: introduce a semaphore in the backend (e.g., `asyncio.Semaphore(N)`) and queue kanban runs. Document an N based on machine size per `hosting.md` (1 GiB / 1 CPU per instance).
5. **Skill discovery rules** ‚Äî Impact: Skills, slash commands, and CLAUDE.md only load when `setting_sources=["project"]` (or `"user"`) is passed ‚Äî forgetting this silently disables every `rapid:*` skill the webUI tries to run. Mitigation: centralise SDK option construction in one helper and unit-test that `setting_sources` contains `"project"`.
6. **Permission-mode default leaks** ‚Äî Impact: SDK default mode still prompts for every non-allowlisted tool; without a UI route for the approval, the session hangs. Mitigation: wire the `can_use_tool` callback to the browser via WebSocket from day one (even for read-only flows), OR pre-populate `allowed_tools` per skill category.
7. **Namespace isolation for RAPID agents** ‚Äî Impact: `agents` option or `AskUserQuestion` payloads could reintroduce non-`rapid:*` tool names. Mitigation: keep `allowed_tools` scoped to the RAPID skill set and validate every `canUseTool` call name against a `rapid:` / `rapid-` / built-in allowlist.
8. **SDK session persistence vs RAPID's `.planning/` state** ‚Äî Impact: SDK stores session state in its own temp dirs (had Windows/APFS leaks pre-0.2.101); RAPID already persists its state under `.planning/STATE.json` with `proper-lockfile`. Mitigation: treat SDK sessions as *ephemeral transport* and continue using RAPID state as the source of truth; persist session IDs in the DB for resume only.
9. **Python `ws://` reverse-proxy** ‚Äî Impact: if the web service is fronted by a reverse proxy, WS upgrade headers must be forwarded, or the agent stream silently closes. Mitigation: document the `Upgrade: websocket` + `Connection: Upgrade` passthrough in deployment docs; uvicorn `[standard]` already handles this locally.
10. **Kanban + autonomous execution feedback loop** ‚Äî Impact: an agent with kanban tools (`add_card`, `move_card`) plus autonomous `/rapid:execute` can queue more work on itself, risking runaway loops. Mitigation: enforce `maxTurns` at the SDK layer AND a per-card execution count ceiling at the kanban service layer; reject recursive card creation from within an agent-run that originated from a card.

## Recommendations (ordered for the roadmap)

1. **Critical ‚Äî Adopt Python SDK, not Node sidecar.** Add `claude-agent-sdk>=0.1.59,<0.2` to `web/backend/pyproject.toml`. Aligns with existing FastAPI stack, zero new runtime. Keep the TS SDK available as an escape hatch if a future need arises, but do not dual-path on day one.
2. **Critical ‚Äî Centralise SDK option construction in one module.** Introduce `web/backend/app/services/agent_sdk.py` that produces `ClaudeAgentOptions` with `setting_sources=["project"]`, `cwd=<project_root>`, allowlisted tools per skill family, and default `max_turns`/`max_budget_usd`. Unit-test every skill route goes through it.
3. **Critical ‚Äî Expose `can_use_tool` over WebSocket from the start.** Even if skills like `/rapid:plan` run without user input, leave the approval channel wired so `/rapid:discuss` and `/rapid:init` work end-to-end. The SDK's `AskUserQuestion` input shape (`{questions: [{question, header, options, multiSelect}]}`) maps cleanly onto a React modal.
4. **High ‚Äî Add custom MCP tools via `create_sdk_mcp_server`/`@tool`.** Kanban mutations (`kanban_add_card`, `kanban_move_card`, `kanban_list_cards`) become in-process MCP tools exposed to agents. This is the SDK's sanctioned tool-registration path; do not invent a parallel mechanism.
5. **High ‚Äî Use V1 `query()`, not V2 preview.** V2 is marked unstable preview and lacks session forking. Revisit when V2 stabilises (track `@anthropic-ai/claude-agent-sdk-python` ‚â•0.2.0 or TS ‚â•0.3.0).
6. **High ‚Äî Persist SDK `session_id` per agent run.** Store in SQLModel alongside kanban cards / agent-run rows to enable resume. Use `forkSession`-style semantics only if the roadmap requires branching experimentation.
7. **High ‚Äî Add concurrency ceiling for agent runs.** Introduce `asyncio.Semaphore` sized to machine resources (default 4, env-tunable). The kanban autonomous-queue will hit this before the SDK does.
8. **Medium ‚Äî Pin bounds, don't exact-pin, for the SDK.** Unlike RAPID core (`zod 3.25.76` exact), the SDK churns weekly ‚Äî use `>=X,<Y` with `<Y` at the next minor. Add a test that imports `claude_agent_sdk.__version__` and asserts the installed version is within the declared range, mirroring the v6.2.0 pin-policy pattern.
9. **Medium ‚Äî Record Claude CLI version on session start.** The `system/init` message includes metadata including `slash_commands` and (since 0.2.105) `memory_paths`. Capture this and display in the "agents" tab ‚Äî makes CLI-drift risk visible.
10. **Medium ‚Äî Gate live-key tests in CI.** Default test suite mocks the SDK (import-level monkeypatch). A single smoke job behind a `CI_LIVE_CLAUDE=1` env var exercises one real `query()` end-to-end. Optional: `pytest-claude-agent-sdk` for LLM-as-judge assertions on higher-risk flows (bug-fix, merge conflict resolution).
11. **Low ‚Äî Tab additions are trivial.** Adding `agents` and `chats` tabs is a pure frontend change against an already-modern stack (React 19 / Vite 8 / Tailwind 4). No new frontend dep needed; reuse TanStack Query for history fetch and a thin WS hook (no `socket.io`) for the live stream. Uses existing CodeMirror for code/diff rendering in messages.
12. **Low ‚Äî Document branding rules.** Anthropic restricts naming: "Claude Code" and "Claude Code Agent" are not permitted for partner products. The webUI tabs should say "Agents" / "Chats" (or "Mission Control Agents"), not "Claude Code". Flag this in the UX research rather than shipping unbranded copy.

## Confidence

- **High confidence** (verified in docs April 2026): SDK package names, version numbers (TS 0.2.109, Py 0.1.59), runtime floors (Node 18+ / Py 3.10+), bundled CLI, `query()` API, `canUseTool` signature, `AskUserQuestion` input shape, V2 preview status, skill discovery via `setting_sources`, plugin option, custom MCP tool APIs.
- **Medium confidence**: concurrency threshold (11+ warning was reported, not stress-tested); CLI version drift impact (plausible from bundled-CLI architecture, not observed).
- **Inferred from codebase**: Python path recommendation over Node sidecar ‚Äî inferred from the existing FastAPI backend, no Node backend in `web/`, and the presence of `uv` tooling. Not "the SDK requires this"; rather, "this matches the existing stack."
