# SET-OVERVIEW: research-agent-reduction

## Approach

The current `/rapid:new-version` skill spawns 6 parallel research agents (stack, features, architecture, pitfalls, oversights, ux) plus a synthesizer -- 7 agent spawns total for the research phase. Analysis of the 6 agents reveals significant domain overlap: "pitfalls" (things that go WRONG) and "oversights" (things people FORGET) are two sides of the same risk/quality coin, and "stack" (dependency versions, tooling) largely feeds into "architecture" (patterns, structure, data flow) with minimal independent value. Meanwhile "ux" covers domain conventions and user expectations which, for a CLI plugin like RAPID, substantially overlaps with "features" (interaction models, API surface, implementation strategies).

The consolidation strategy is to merge overlapping pairs into broader agents while preserving full domain coverage. The target is 3 consolidated research agents plus the synthesizer (4 agent spawns total, down from 7). The 3 consolidated agents would cover: (1) stack-and-architecture (technology + patterns + structure), (2) features-and-ux (implementation + user expectations + domain conventions), (3) risks-and-quality (pitfalls + oversights + cross-cutting concerns). The synthesizer prompt must then be updated to expect 3 input files instead of 6, with adjusted section mappings.

All changes are confined to `skills/new-version/SKILL.md` -- the agent definition files under `agents/` are generated by the build pipeline and are not directly edited. The SKILL.md contains the full prompt text for each research agent spawn inline in Step 5, so the consolidation is purely a SKILL.md rewrite of the agent spawn prompts, the file list in Step 6 (synthesizer input), and the anti-pattern rules at the bottom.

## Integration Points

- **Exports:** Reduced research pipeline in SKILL.md -- fewer than 6 agents spawned with equivalent domain coverage
- **Imports:** None -- this set has no dependencies on other sets
- **Side Effects:** The synthesizer agent (`rapid-research-synthesizer`) receives fewer input files (3 instead of 6). The output format (SUMMARY.md) and the downstream consumer (`rapid-roadmapper`) are unchanged. The SKILL.md description line and anti-pattern rules must be updated to reflect the new agent count.

## Risks

| Risk | Impact | Mitigation |
|------|--------|------------|
| Coverage gaps from merging agents -- consolidated prompts may lose specific research areas that were explicit in the original agents | High | Map every output section from the 6 original agents to a section in the 3 consolidated prompts; verify 1:1 coverage via checklist |
| Synthesizer format breakage -- the synthesizer expects 6 named files (STACK.md, FEATURES.md, etc.) and its role prompt references them by name | High | Update the synthesizer prompt in Step 6 to reference the 3 new filenames and adjust its deduplication logic accordingly |
| Prompt bloat -- merging two agents into one doubles the prompt size per agent, potentially degrading research quality | Medium | Keep consolidated prompts concise by eliminating redundant boilerplate (shared identity blocks, overlapping scope exclusion lists) |
| Anti-pattern rule staleness -- the "Anti-Patterns" section at the bottom of SKILL.md mandates "MUST spawn all 6" | Medium | Update the anti-pattern rules and description field to reflect the new count |
| Build pipeline agent definitions become orphaned -- the `agents/rapid-research-*.md` files for merged agents remain but are no longer spawned | Low | Leave agent files in place (they are generated artifacts); document that they are no longer used by new-version |

- **Wave 2:** Synthesizer and housekeeping -- update Step 6 synthesizer prompt for 3 inputs, update SKILL.md description line, update anti-pattern rules, update the "Important Constraints" section
