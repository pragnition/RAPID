<objective>
Wire SKILL.md and commands/init.md to call rapid-tools.cjs init scaffold instead of using Write tool with inline templates.

Purpose: The verification report (02-VERIFICATION.md) found that SKILL.md Step 5 uses the Write tool with inline templates to create .planning/ files, bypassing the init.cjs library entirely. This gap closure wires the intended integration path: SKILL.md instructs Claude to call the CLI, which dispatches to init.cjs.

Output: Updated SKILL.md and commands/init.md where Step 5 calls `rapid-tools.cjs init scaffold` and Step 3 calls `rapid-tools.cjs init detect`, eliminating all inline template definitions. No code changes -- only Markdown skill/command file updates.
</objective>
