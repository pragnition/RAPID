  <behavior>
    - detectCodebase(cwd) returns { hasSourceCode: false, languages: [], frameworks: [], configFiles: [], sourceStats: {} } for empty directory
    - detectCodebase(cwd) returns hasSourceCode: false for directory with only .planning/ and .git/
    - detectCodebase(cwd) returns hasSourceCode: true and languages includes 'javascript' when package.json exists
    - detectCodebase(cwd) returns hasSourceCode: true and languages includes 'typescript' when tsconfig.json exists
    - detectCodebase(cwd) detects go.mod (go), Cargo.toml (rust), pyproject.toml (python), requirements.txt (python), Gemfile (ruby), pom.xml (java), build.gradle (java)
    - detectCodebase(cwd) detects frameworks: React (react in package.json deps), Express (express in deps), Next.js (next in deps), Vue (vue in deps), Django (django in requirements.txt or pyproject.toml)
    - detectCodebase(cwd) populates sourceStats with counts of source files by extension (only checks top 3 directory levels to avoid node_modules depth)
    - detectCodebase(cwd) skips node_modules, .git, .planning, vendor, dist, build, __pycache__, .venv directories during source file counting
    - detectConfigFiles(cwd) finds .eslintrc* files, eslint.config.* files, .prettierrc* files, prettier.config.* files
    - detectConfigFiles(cwd) finds tsconfig*.json, .editorconfig, jest.config.*, vitest.config.*, .mocharc.*, pytest.ini
    - detectConfigFiles(cwd) finds .github/workflows/*.yml, .gitlab-ci.yml, Jenkinsfile
    - detectConfigFiles(cwd) finds .husky/*, .pre-commit-config.yaml
    - detectConfigFiles(cwd) parses JSON config files (.json extension) and returns parsed content; returns raw text path for non-JSON configs
    - detectConfigFiles(cwd) returns array of { file, category, parsed } objects where category is one of: linting, formatting, typescript, editor, testing, ci, git-hooks
    - mapDirectoryStructure(cwd, maxDepth) returns tree limited to specified depth (default 3)
    - mapDirectoryStructure skips node_modules, .git, .planning, vendor, dist, build, __pycache__, .venv
    - mapDirectoryStructure returns objects with { name, type: 'file'|'directory', children? } structure
    - buildScanManifest(cwd) calls detectCodebase, detectConfigFiles, and mapDirectoryStructure and returns combined { codebase, configFiles, structure, sampleFiles } object
    - buildScanManifest selects up to 10 sample source files per language, prioritizing entry points (index.*, main.*, app.*), config files, and test files
    - buildScanManifest returns sampleFiles as array of { path, language, priority } objects
  </behavior>
  <action>
    Create `rapid/src/lib/context.cjs` as a CommonJS module following the established library pattern (like init.cjs, core.cjs).

    **detectCodebase(cwd):**
    - Check for language-specific manifest files: package.json, tsconfig.json, go.mod, Cargo.toml, pyproject.toml, requirements.txt, Gemfile, pom.xml, build.gradle, build.gradle.kts, Makefile, CMakeLists.txt
    - For package.json: parse JSON and check dependencies/devDependencies for framework indicators (react, express, next, vue, angular, fastify, koa, nest)
    - For pyproject.toml/requirements.txt: read as text and check for framework names (django, flask, fastapi)
    - Count source files by extension (.js, .ts, .jsx, .tsx, .py, .go, .rs, .rb, .java, .kt, .swift, .c, .cpp, .cs) using fs.readdirSync({ recursive: true }) limited to 3 levels deep
    - Skip directories: node_modules, .git, .planning, vendor, dist, build, __pycache__, .venv, .next, coverage, .cache
    - Return: { hasSourceCode, languages, frameworks, configFiles (manifest filenames), sourceStats: { '.js': N, '.ts': N, ... } }

    **detectConfigFiles(cwd):**
    - Use fs.readdirSync for top-level and specific subdirectories (.github/workflows/, .husky/)
    - For each found config file, categorize into: linting, formatting, typescript, editor, testing, ci, git-hooks
    - For .json extension files, parse with JSON.parse in try/catch (graceful failure returns null for parsed)
    - Return: array of { file, category, parsed } where parsed is the JSON object or null

    **mapDirectoryStructure(cwd, maxDepth = 3):**
    - Recursive directory listing with depth limit
    - Skip directories in the exclusion list
    - Return tree structure: { name, type: 'file'|'directory', children?: [] }

    **buildScanManifest(cwd):**
    - Orchestrate the three detection functions
    - Select sample source files: up to 10 per detected language
    - Prioritize entry points (index.*, main.*, app.*, server.*), then test files (*.test.*, *.spec.*), then one file from each top-level source directory
    - Return: { codebase, configFiles, structure, sampleFiles: [{ path, language, priority }] }

    Export all four functions. Use `require('fs')` and `require('path')` only (zero new dependencies).
  </action>
