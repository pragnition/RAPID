    - createInitialState('my-project', 'v2.0') returns a valid ProjectState with version:1, empty milestones with one milestone entry, timestamps set
    - readState(cwd) returns { valid: true, state } for valid STATE.json, { valid: false, errors } for invalid, null if file missing
    - writeState(cwd, state) acquires lock, validates with ProjectState.parse, updates lastUpdatedAt, writes to .tmp then renames to STATE.json, releases lock
    - writeState rejects invalid state with ZodError (does not write to disk)
    - findMilestone(state, milestoneId) returns milestone or throws
    - findSet(state, milestoneId, setId) returns set or throws
    - findWave(state, milestoneId, setId, waveId) returns wave or throws
    - findJob(state, milestoneId, setId, waveId, jobId) returns job or throws
    - Find helpers throw descriptive errors like "Set 'foo' not found in milestone 'bar'"
    - transitionJob(cwd, milestoneId, setId, waveId, jobId, newStatus) validates transition, updates job status, sets timestamps, derives wave status, writes state
    - transitionJob sets startedAt when moving to 'executing', sets completedAt when moving to 'complete' or 'failed'
    - transitionWave and transitionSet validate and update status directly (for manual orchestration steps like 'discussing' > 'planning')
    - deriveWaveStatus(jobs) returns: 'pending' if all pending, 'complete' if all complete, 'failed' if any failed and none executing, 'executing' otherwise
    - deriveSetStatus(waves) returns: 'pending' if all pending, 'complete' if all complete, 'executing' if any wave is in an active state
    - detectCorruption(cwd) returns { exists, corrupt?, reason?, errors? }
    - recoverFromGit(cwd) runs `git checkout HEAD -- .planning/STATE.json`
    - commitState(cwd, message) runs `git add .planning/STATE.json && git commit -m "{message}"`
