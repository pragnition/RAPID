## Decisions Made
- **Role maps inline in handleBuildAgents:** Rather than creating a new module, all maps (ROLE_TOOLS, ROLE_COLORS, ROLE_DESCRIPTIONS, ROLE_CORE_MAP) and assembly functions are scoped inside handleBuildAgents. - **No agents default in loadConfig:** Since config.json no longer has an agents section, the loadConfig defaults in core.cjs were cleaned to not include it either.
- **Test strategy shift:** build-agents.test.cjs now tests the generated output directly (reads from agents/ directory) instead of testing assembler module functions. This is more robust since it validates the end-to-end pipeline.
