**Section 6: Key Concepts**
Brief explanations (2-3 sentences each) of:
- **Sets**: Independent workstreams that run in isolated git worktrees
- **Interface Contracts**: Machine-verifiable JSON schemas defining API surfaces between sets
- **Waves**: Dependency-ordered execution groups (Wave 1 runs first, Wave 2 depends on Wave 1)
- **Planning Gates**: Enforcement that all sets must be planned before execution begins
- **Wave Reconciliation**: Mandatory contract validation between execution waves
- **Ownership**: Every file belongs to exactly one set; cross-set changes use CONTRIBUTIONS.json
