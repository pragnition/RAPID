<feature>
  <name>Bisection recovery, rollback, and agent integration</name>
  <files>src/lib/merge.cjs, src/lib/merge.test.cjs</files>
  <behavior>
    ## Bisection Recovery (MERG-05)

    - bisectWave(cwd, baseBranch, mergedSets, preWaveCommit): Performs binary search over merged sets to find the breaking set interaction.
      1. Saves .planning/ directory to temp location (fs.cpSync or tar) to preserve MERGE-STATE.json
      2. Runs git reset --hard to preWaveCommit
      3. Binary search loop: splits mergedSets, re-merges subset via mergeSet(), runs runIntegrationTests()
      4. After each iteration, resets back to preWaveCommit
      5. When breaking set identified, restores .planning/ from temp
      6. Updates MERGE-STATE.json bisection field for the breaking set
      7. Returns {breakingSet: string, iterations: number, testOutput: string}
    - getPreWaveCommit(cwd): Gets current HEAD commit hash before wave merging starts, stores for later bisection use. Returns string.

    Test cases:
    - bisectWave with single set returns that set as breaking (trivial case)
    - bisectWave with two sets identifies the one that causes test failure
    - bisectWave preserves .planning/ directory after completion
    - bisectWave updates MERGE-STATE.json with bisection results
    - getPreWaveCommit returns current HEAD hash

    ## Single-Set Rollback (MERG-06)

    - revertSetMerge(cwd, setId): Reads MERGE-STATE.json to get mergeCommit hash, executes git revert -m 1 --no-edit on that commit. Returns {reverted: boolean, revertCommit?: string, reason?: string, detail?: string}
    - detectCascadeImpact(cwd, setId): Reads DAG.json, finds sets that depend on setId AND have already been merged (mergeStatus in MERGE-STATE or REGISTRY). Returns {hasCascade: boolean, affectedSets: string[], recommendation: string}
    - When revertSetMerge encounters a conflict during revert, it returns {reverted: false, reason: 'conflict', detail: ...}

    Test cases:
    - revertSetMerge successfully reverts a merge commit
    - revertSetMerge returns error when mergeCommit missing in MERGE-STATE.json
    - revertSetMerge handles conflict during revert gracefully
    - detectCascadeImpact returns hasCascade=false for set with no dependents
    - detectCascadeImpact returns hasCascade=true with affected sets for set with merged dependents
    - detectCascadeImpact returns hasCascade=false when dependents haven't merged yet

    ## Agent Integration (MERG-01, MERG-02 completion)

    - integrateSemanticResults(detectionResults, agentResults): Merges merger agent's semantic conflict findings into the detection report's semantic field. Returns updated detectionResults.
    - applyAgentResolutions(resolutions, agentResults, confidenceThreshold): Takes agent's resolutions, applies those above threshold (marks as tier 3), marks those below threshold for human escalation (tier 4). Returns [{conflict, tier: 3|4, resolved, confidence, resolution?, escalation?}]
    - Default confidenceThreshold = 0.7 (configurable)

    Test cases:
    - integrateSemanticResults merges agent findings into detection.semantic
    - applyAgentResolutions marks high-confidence resolutions as tier 3 applied
    - applyAgentResolutions escalates low-confidence resolutions to tier 4
    - applyAgentResolutions uses default threshold of 0.7
    - applyAgentResolutions respects custom threshold
  </behavior>
  <implementation>
    Add new functions to src/lib/merge.cjs (appending to the module written in Plan 01):

    **Bisection:**
    1. bisectWave: Use fs.cpSync (Node 16.7+) to copy .planning/ to os.tmpdir() before git operations. Use execFileSync for git reset --hard. Binary search with lo/hi pointers. Try/finally to restore .planning/ from temp. Use updateMergeState to record bisection results.
    2. getPreWaveCommit: Simple wrapper around git rev-parse HEAD.

    **Rollback:**
    1. revertSetMerge: Read MERGE-STATE.json for mergeCommit. Use execFileSync for git revert -m 1 --no-edit. Catch CONFLICT errors. Return structured result matching mergeSet pattern.
    2. detectCascadeImpact: Read DAG.json for edges. For each edge where setId is a dependency, check if the dependent set has status='complete' in MERGE-STATE. Collect affected sets.

    **Agent Integration:**
    1. integrateSemanticResults: Deep-copy detection results, set .semantic field from agent data. Simple merge.
    2. applyAgentResolutions: Iterate agent resolutions, compare confidence to threshold, categorize as tier 3 (resolved) or tier 4 (escalated).

    **Critical pitfalls to handle:**
    - git reset --hard during bisection destroys .planning/ -- MUST save/restore (Pitfall 3 from RESEARCH.md)
    - git revert on merge commit MUST use -m 1 -- parent 1 is the base branch (Pitfall 2 from RESEARCH.md)
    - After rollback, run integration tests immediately -- don't cascade automatically (Pitfall 4)

    Add all new functions to module.exports.

  </implementation>
