  <name>Task 1: Job-level reconciliation, progress banners, and job-level handoff in execute.cjs</name>
  <files>src/lib/execute.cjs, src/lib/execute.test.cjs</files>
  <behavior>
    - reconcileJob(cwd, setId, waveId, jobId, worktreePath, baseBranch): returns { passed, failed } with file_exists/missing_file and commit_format checks
    - reconcileJob with missing files returns failed entries with type 'missing_file'
    - reconcileJob with valid commit messages returns passed entries with type 'commit_format_valid'
    - reconcileJob with invalid commit messages returns failed entries with type 'commit_format_violation'
    - reconcileWaveJobs(cwd, setId, waveId, worktreePath, baseBranch): aggregates per-job results, returns { hardBlocks, softBlocks, jobResults, overall }
    - reconcileWaveJobs returns 'PASS' when all jobs pass
    - reconcileWaveJobs returns 'PASS_WITH_WARNINGS' when only soft blocks exist
    - reconcileWaveJobs returns 'FAIL' when hard blocks exist
    - formatProgressBanner(waveId, waveIndex, totalWaves, jobStatuses, timestamp): returns formatted banner string with wave header + indented job lines
    - formatProgressBanner includes timestamp in [HH:MM] format
    - generateJobHandoff(checkpointData, setId, waveId, jobId, pauseCycle): returns Markdown with frontmatter including set/wave/job context
    - parseJobHandoff(content): returns structured object with set, waveId, jobId, pauseCycle, completedWork, remainingWork, resumeInstructions
  </behavior>
  <action>
    Extend execute.cjs with the following new functions (do NOT modify existing functions):

    1. **reconcileJob(cwd, setId, waveId, jobId, worktreePath, baseBranch)**: Verify a single job's deliverables against its JOB-PLAN.md. Read the job plan from `.planning/waves/{setId}/{waveId}/{jobId}-PLAN.md`. Parse "Files to Create/Modify" entries from the plan. Check file existence in worktreePath. Check commit format using `type(setId): description` pattern (reuse existing `escapeRegExp` and `getCommitMessages`). Return `{ passed: [{type, target}], failed: [{type, target}] }`.

    2. **reconcileWaveJobs(cwd, setId, waveId, worktreePath, baseBranch)**: Aggregate per-job reconciliation. Read all `*-PLAN.md` files from `.planning/waves/{setId}/{waveId}/`. Call reconcileJob for each. Compute jobResults map (filesPlanned, filesDelivered, missingFiles, commitViolations per job). Compute hardBlocks (none for now, reserved for future test-based checks), softBlocks (missing_file, commit_format_violation). Return `{ hardBlocks, softBlocks, jobResults, overall }` where overall is PASS/PASS_WITH_WARNINGS/FAIL.

    3. **formatProgressBanner(waveId, waveIndex, totalWaves, jobStatuses, timestamp)**: Generate a formatted progress banner. jobStatuses is an array of `{ jobId, status, tasksCompleted, tasksTotal }`. Format:
       ```
       --- RAPID Execute ---
       Wave {waveId} ({waveIndex}/{totalWaves})
         {jobId}: {status} ({tasksCompleted}/{tasksTotal} steps)
         ...
       [{timestamp}]
       ---------------------
       ```
       Status can be: Executing, Complete, Failed, Pending, Paused.

    4. **generateJobHandoff(checkpointData, setId, waveId, jobId, pauseCycle)**: Like existing generateHandoff but with wave/job context in frontmatter. Stores at `.planning/waves/{setId}/{waveId}/{jobId}-HANDOFF.md`.

    5. **parseJobHandoff(content)**: Like existing parseHandoff but extracts waveId, jobId from frontmatter.

    Add all five functions to module.exports. Write comprehensive tests in execute.test.cjs using the existing test patterns (node:test describe/it, node:assert/strict, tmp directory fixtures).

    For reconcileJob tests: create a tmp directory with mock JOB-PLAN.md files containing "Files to Create/Modify" sections, create/omit actual files to test existence checks, mock git commit messages.

    For progress banner tests: verify output format matches the banner pattern.

    For job handoff tests: verify round-trip (generate then parse).
  </action>
