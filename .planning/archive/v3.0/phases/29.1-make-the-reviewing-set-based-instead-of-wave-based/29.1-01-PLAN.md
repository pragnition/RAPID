<feature>
  <name>Set-level review library with TDD</name>
  <files>src/lib/review.cjs, src/lib/review.test.cjs</files>
  <behavior>
    ## Schema Changes
    - ReviewIssue.parse({...originatingWave: 'wave-1'}) succeeds (optional field)
    - ReviewIssue.parse({...without originatingWave}) succeeds (backward compat)
    - ReviewIssues.parse({setId, issues, lastUpdatedAt}) succeeds WITHOUT waveId
    - ReviewIssues.parse({waveId, setId, issues, lastUpdatedAt}) FAILS (waveId removed)

    ## scopeSetForReview (rename)
    - scopeSetForReview(cwd, worktreePath, baseBranch) returns {changedFiles, dependentFiles, totalFiles}
    - scopeWaveForReview is no longer exported (removed from module.exports)

    ## chunkByDirectory (new)
    - Files <= 15 (CHUNK_THRESHOLD) -> returns [{dir: '.', files: allFiles}] (single chunk, no splitting)
    - Files > 15 across 3 directories -> returns 3 chunks, one per directory
    - Small directories (< 3 files) merge into the last large chunk
    - All files in one directory but > 15 -> returns [{dir: 'that/dir', files: allFiles}] (single chunk)
    - After chunking, if only 1 chunk results, still returns that 1 chunk (caller decides parallelism)
    - Empty files array -> returns [{dir: '.', files: []}]

    ## buildWaveAttribution (new)
    - Reads JOB-PLAN.md files from .planning/waves/{setId}/{waveId}/ directories
    - Extracts file paths from "Files to Create/Modify" table using regex
    - Returns map: {filePath -> waveId}
    - Last wave wins when file appears in multiple waves
    - Returns empty object when wavesDir does not exist
    - Skips malformed plan files gracefully

    ## logIssue (refactored)
    - logIssue(cwd, setId, issue) writes to .planning/waves/{setId}/REVIEW-ISSUES.json (NOT wave subdirectory)
    - Issue can have originatingWave field (optional)
    - Container JSON has {setId, issues, lastUpdatedAt} -- no waveId
    - Appends to existing file without losing prior issues

    ## loadSetIssues (simplified)
    - Reads single .planning/waves/{setId}/REVIEW-ISSUES.json
    - Returns flat array of issues with originatingWave preserved from each issue
    - Returns empty array when file does not exist
    - Falls back to reading wave subdirectories for legacy compatibility (wave-level REVIEW-ISSUES.json from lean review)

    ## updateIssueStatus (simplified)
    - updateIssueStatus(cwd, setId, issueId, newStatus) -- no waveId parameter
    - Reads from .planning/waves/{setId}/REVIEW-ISSUES.json
    - Sets fixedAt timestamp when newStatus is 'fixed'
    - Throws when issue not found

    ## generateReviewSummary (minor update)
    - Groups by originatingWave instead of waveId for per-wave breakdown
    - Still produces valid markdown summary
  </behavior>
