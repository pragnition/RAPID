  <name>Task 1: Status formatting and scoped CLAUDE.md generation</name>
  <files>rapid/src/lib/worktree.cjs, rapid/src/lib/worktree.test.cjs, rapid/src/bin/rapid-tools.cjs</files>
  <behavior>
    - formatStatusTable(worktrees) renders ASCII table with columns SET, BRANCH, PHASE, STATUS, PATH; column widths auto-calculated; header + separator + rows
    - formatStatusTable([]) returns a "No active worktrees" message
    - formatWaveSummary(registry, dagJson) produces per-wave summary lines like "Wave 1: 2/3 done, 1 executing" or "Wave 2: 3 sets pending"
    - formatWaveSummary with no DAG data returns empty string
    - generateScopedClaudeMd(cwd, setName) produces a self-contained Markdown string containing: set name header, scope description, interface contract JSON block, owned files list, DO NOT TOUCH deny list from OWNERSHIP.json, and style guide content if available
    - generateScopedClaudeMd deny list includes all files from OWNERSHIP.json NOT owned by the target set
    - `rapid-tools worktree status` outputs formatted table (human-readable) with wave summary, or JSON with --json flag
  </behavior>
  <action>
    Write tests first in worktree.test.cjs, then implement in worktree.cjs.

    **formatStatusTable(worktrees):**
    - Accept array of worktree entry objects (from registry): `[{ setName, branch, phase, status, path }]`
    - Headers: `['SET', 'BRANCH', 'PHASE', 'STATUS', 'PATH']`
    - Calculate column widths as max(header.length, max(row values lengths))
    - Pad each cell with `.padEnd(width)`, join columns with `'  '` (two spaces)
    - Separator line: dashes `'-'.repeat(width)` joined with `'  '`
    - Return `[headerLine, separator, ...rowLines].join('\n')`
    - If worktrees array is empty, return `'No active worktrees. Run worktree create to get started.'`

    **formatWaveSummary(registry, dagJson):**
    - If dagJson is null or missing waves, return `''`
    - For each wave in dagJson.waves (sorted by wave number):
      - Count total sets in wave
      - Count sets with phase 'Done' in registry
      - Count sets with phase 'Executing' in registry
      - Count sets with phase 'Error' in registry
      - Format line: `Wave {N}: {done}/{total} done{, {active} executing if active > 0}{, {error} error if error > 0}` or `Wave {N}: {total} sets pending` if none active/done
    - Return lines joined with `'\n'`

    **generateScopedClaudeMd(cwd, setName):**
    - Load set via `plan.loadSet(cwd, setName)` -- requires plan.cjs
    - Read OWNERSHIP.json from `.planning/sets/OWNERSHIP.json` (graceful: return reduced content if missing)
    - Read style guide from `.planning/context/STYLE_GUIDE.md` if it exists (graceful: skip section if missing)
    - Build deny list: filter OWNERSHIP.json entries where owner !== setName, collect file paths
    - Build owned files list: filter OWNERSHIP.json entries where owner === setName, collect file paths
    - Assemble sections:
      1. `# Set: {setName} -- Scoped Agent Context` header
      2. `## Your Scope` -- "You are working on the '{setName}' set. ONLY modify files listed under File Ownership."
      3. `## Interface Contract` -- CONTRACT.json content in fenced JSON block
      4. `## File Ownership` -- "You may ONLY modify these files:" + bullet list of owned files
      5. `## DO NOT TOUCH` -- "These files are owned by other sets. Do NOT modify them:" + bullet list of deny files (grouped by owning set)
      6. `## Style Guide` -- style guide content (if available)
    - Return the assembled Markdown string

    **CLI `worktree status` subcommand:**
    - Add `status` to the handleWorktree switch in rapid-tools.cjs
    - Call `reconcileRegistry(cwd)` to ensure freshness
    - If `--json` flag present: output JSON `{ worktrees: [...], waves: waveSummaryObj }`
    - Otherwise: Read DAG.json from `.planning/sets/DAG.json` (graceful if missing), call formatWaveSummary and formatStatusTable, write human-readable output to stdout

    Also add `generate-claude-md` subcommand:
    - `rapid-tools worktree generate-claude-md <set-name>` -- generates and writes scoped CLAUDE.md to the worktree's root directory
    - Read registry to find worktree path for set
    - Call generateScopedClaudeMd(cwd, setName)
    - Write result to `{worktreePath}/CLAUDE.md`
    - Output JSON `{ generated: true, setName, path: worktreePath + '/CLAUDE.md' }`

    **Update USAGE string** in rapid-tools.cjs to include:
    ```
    worktree status             Show all worktrees with status table
    worktree status --json      Machine-readable worktree status
    worktree generate-claude-md <set>  Generate scoped CLAUDE.md for a worktree
    ```
