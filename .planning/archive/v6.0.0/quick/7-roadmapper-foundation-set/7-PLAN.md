# Quick Task 7: Teach Roadmapper to Include Foundation Set

## Objective

Move foundation set creation from a bolted-on Step 9a in `skills/init/SKILL.md` into the roadmapper's native output. When `team-size > 1`, the roadmapper will include a foundation set at index 0 in its `state.milestones[].sets[]` array and a corresponding contract entry. This eliminates the post-roadmap patching that Step 9a attempted (and failed due to context loss).

## Context

- `createFoundationSet()` in `src/lib/scaffold.cjs` and the `scaffold create-foundation-set` CLI subcommand remain untouched -- they still work for programmatic use.
- The foundation set merges all exports from all other sets into a single contract (same logic as `createFoundationSet` but done declaratively by the roadmapper agent in its JSON output).
- Solo projects (`team-size = 1`) must NOT get a foundation set.

---

## Task 1: Add Foundation Set Instructions to Roadmapper Role Module

**Files:** `src/modules/roles/role-roadmapper.md`

**Action:**

Add a new `## Foundation Set (Multi-Developer Only)` section to the role module, placed after the existing `## Group Assignment Guidance` section and before `## Scope and Constraints`. This section instructs the roadmapper to:

1. When `team-size > 1`, include a foundation set as the **first entry** (index 0) in `state.milestones[].sets[]` with these exact fields:
   - `id: "foundation"`
   - `name: "Foundation"`
   - `status: "pending"`
   - `branch: "set/foundation"`
   - No `waves` array (foundation sets are not executed via wave/job pipeline)

2. Include a corresponding entry in the `contracts` array with:
   - `setId: "foundation"`
   - `contract` object containing:
     - `foundation: true`
     - `exports` that merge ALL exports from ALL other sets (each export entry includes a `sourceSet` field indicating which set originally exports it)
     - Empty `imports` (foundation set imports nothing)
     - `definition.scope: "Foundation set containing shared interfaces and stubs for multi-group parallel development."`
   - `definition` object with:
     - `scope: "Foundation set containing shared interfaces and stubs for multi-group parallel development. This set must not contain feature implementation logic."`
     - `ownedFiles: []` (foundation set owns no files exclusively)
     - `tasks: []` (no implementation tasks)
     - `acceptance: ["All cross-set interface stubs are present and importable"]`

3. Include the foundation set in the ROADMAP.md markdown output as "Set 0: Foundation" before all other sets, with scope description and "Dependencies: none".

4. When `team-size = 1`, do NOT include a foundation set. The output remains unchanged from current behavior.

Additionally, update the `### STATE.json Structure` example in the Output section to show the foundation set at index 0 with a comment indicating it is conditional on `team-size > 1`.
