# RAPID Agent Prompt Schema

## Overview

Agent prompts are assembled from XML-tagged sections containing Markdown content. Each tag wraps a distinct concern -- identity, role instructions, tool reference, return protocol, conventions, or runtime context. This schema defines the allowed tags and their semantics.

This document is a build-time reference. It is NOT embedded in agent prompts.

## Tags

### Required Tags

Every built agent must include these tags. The build pipeline warns if any are missing.

| Tag | Purpose | Source |
|-----|---------|--------|
| `<identity>` | Agent identity, workflow, tool invocation, context loading, state rules | `core-identity.md` |
| `<role>` | Role-specific instructions and behavioral guidance | `role-{name}.md` |
| `<returns>` | Structured return protocol (COMPLETE, CHECKPOINT, BLOCKED) | `core-returns.md` |

### Optional Tags

These tags are included only when applicable to the role.

| Tag | Purpose | Source |
|-----|---------|--------|
| `<conventions>` | Git commit conventions for roles that commit code | `core-conventions.md` |
| `<tools>` | Per-role CLI command reference (compact YAML) | Generated by `tool-docs.cjs` via `getToolDocsForRole()` |
| `<context>` | Runtime task-specific context (plan details, state, files) | Injected by skills at spawn time |

## Assembly Order

Built agent prompts follow this section order:

1. **YAML frontmatter** -- name, description, tools, model, color
2. **`<identity>`** -- always first content section
3. **`<conventions>`** -- if role commits code (omitted otherwise)
4. **`<tools>`** -- if role uses CLI commands (omitted otherwise)
5. **`<role>`** -- role-specific instructions
6. **`<returns>`** -- always last static section

Note: `<context>` is injected at runtime by the spawning skill, not during the build step. It appears after the static sections when the agent is invoked.

## Rules

- All tags are top-level -- no nesting
- Content inside tags is Markdown (headings, lists, code blocks, etc.)
- Optional tags may be omitted entirely if the role does not need them
- Required tags (`identity`, `role`, `returns`) must be present in every built agent -- build emits a warning if missing
- Tag order in the assembled prompt follows the assembly order above
- The schema itself is never embedded in agent prompts -- it exists only as a reference for the build pipeline and developers
