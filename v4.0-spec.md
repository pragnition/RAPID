We will be adding a rather complex dashboard view to RAPID.
This dashboard view will function as a "management" ui where users can look at running projects and do knowledge management from.

This webservice will live at ~/.rapid/ and should start on boot (be registered as a service).
The webserver will be hosted on port 8998. 

This webserver will serve as a registry of active RAPID projects that the user is working on.
It will include a knowledge graph of each project, the state of the project (sets, status of set, versions etc) and a kanban board + markdown note editor for users to write todo lists and track their progress.
Also include a codebase mapping system for codebases that are compatible

it should also track active worktrees etc.

Therefore, the webserver is _not_ meant as a means for the user to edit/work on the project in anyway, it is just a knowledge management platform, basically. Features to edit the project might come later, but not now.



## Install  

Naturally, to run on startup, rapid has to be registered with init/systemd/whatever init binary the os uses.
Claude code might not be running with elevated permissions. Therefore, the install command should tell the user how to setup the service in order to keep RAPID always running.

This should also be an OPTIONAL feature. Prompt the user and ask them if they want to enable it during /rapid:install.

The way future commands should detect if the web service is enabled (eg. for the /init command) is to check the env vars for the prescence of RAPID_WEB=true. This env var will be installed to the users shell config similarly to how the RAPID_TOOLS is installed.

## Registering a project

When the /init command is ran, it should detect for the RAPID_WEB environment variable, and then attempt to register the project with the rapid management interface. The way we shall do this is to include a function in rapid_tools that will aid with this.

For projects using old rapid versions, we should have a /register_web command to register the old project to the service.
